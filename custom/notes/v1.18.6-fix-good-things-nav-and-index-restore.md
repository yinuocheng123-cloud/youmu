# 版本记录：v1.18.6-fix-good-things-nav-and-index-restore

## 1. 本轮目标

本轮目标是修复 `146c061 V1.18.6 好物首页编号移除+更多入口+全站同步文章流` 中出现的误伤问题。该提交已经推送，但验证中暴露 `content-depth`、`release-readiness`、`navigation-consistency` 失败，并且 `index.html`、`solutions/index.html`、`data/site-content.js` 存在编码损坏、Header 导航结构被破坏、好物下拉和六类体系丢失等问题。

本轮属于 Bug 修复和发布阻塞清理。核心目标是恢复完整 Header、桌面端与移动端好物下拉、好物首页六类体系，并保留 V1.18.6 正确需求：去掉好物卡片中的 01-06 编号视觉元素，并给六类入口补“继续阅读”入口。

## 2. 背景与上下文

本轮开始时先执行了 `git status`、`git pull --ff-only` 和 `git log --oneline -5`。当前分支为 `master`，远程已是最新，最近提交为 `146c061 V1.18.6 好物首页编号移除+更多入口+全站同步文章流`。

历史记录中最新正式记录仍为 `custom/notes/v1.18.5-good-things-final-content-quality-polish.md`，未发现 V1.18.6 对应正式记录。V1.18.5 的关键基线是：柚木好物首页必须保留六类体系，30 个好物文章页、外部资料、站内关联和公开旧词拦截均应保持稳定。

本轮不继续沿用此前误伤来源的 PowerShell 正则替换思路，而是回到 V1.18.5 的稳定文件，再做局部精确修改。

## 3. 问题分析

表面问题是多个检查脚本失败：

- `data/site-content.js` 存在字符串损坏，无法通过 `node --check`。
- `index.html` 和 `solutions/index.html` 无法通过严格 UTF-8 解码。
- 好物下拉中六类入口被破坏，导航一致性检查失败。
- `solutions/index.html` 中六类体系无法被内容深度检查识别。

核心原因是 V1.18.6 中对 HTML 做了不够精确的批量替换，导致页面结构和中文编码被误伤。该问题不是单个链接缺失，而是公开页基础结构被破坏，因此需要先恢复稳定基线，再手工补正确需求。

## 4. 候选方案比较

方案一：在 `146c061` 损坏文件上继续修补。

优点是理论上能保留 V1.18.6 的全部差异；缺点是文件已经存在编码损坏和结构误伤，继续在损坏基础上修补很容易遗漏问题。

方案二：从 `4f204fc` 恢复 `index.html`、`solutions/index.html`、`data/site-content.js`、`styles.css`，再手工应用 V1.18.6 的正确意图。

优点是可以回到已验证通过的 V1.18.5 稳定基线；缺点是需要重新确认 V1.18.6 的正确需求是否都补回。最终选择此方案。

方案三：直接回滚整个 V1.18.6。

优点是最简单；缺点是会丢掉“去编号”和“增加入口”的需求，也没有形成修复提交。

## 5. 最终决策

最终采用方案二：

- 从 `4f204fc` 恢复 `index.html`、`solutions/index.html`、`data/site-content.js`、`styles.css`。
- 保留全站 Header 主导航：`首页`、`认识柚喜`、`柚木知识`、`柚木好物`、`推荐厂商`、`社群交流`。
- 恢复桌面端和移动端 `柚木好物` 下拉：`柚木好物首页`、`柚木家具`、`柚木地板`、`柚木整装`、`柚木户外`、`柚木收藏`、`柚木文创`。
- 将文创锚点按本轮要求统一为 `good-cultural`，并同步检查脚本。
- 删除首页好物卡片和好物首页内容卡片中的 01-06 编号视觉文本，但保留卡片和视觉块。
- 给 `solutions/index.html` 六类入口卡补“继续阅读”链接。

## 6. 具体实现

本轮恢复或修改的重点文件：

- `index.html`：从 `4f204fc` 恢复完整 Header、移动端菜单、首页结构；删除好物卡片视觉编号；将文创下拉锚点同步到 `good-cultural`。
- `solutions/index.html`：从 `4f204fc` 恢复完整 Header、移动端菜单、六类体系和 30 个好物入口；删除六类入口卡和内容卡中的编号视觉文本；补充六类入口卡“继续阅读”链接；将文创区块 id 改为 `good-cultural`。
- `data/site-content.js`：从 `4f204fc` 恢复可解析 UTF-8 文件，并同步文创锚点到 `good-cultural`。
- `styles.css`：从 `4f204fc` 恢复稳定样式，避免保留误伤版本的样式侧效应。
- 全站公开 HTML：同步 Header 中 `柚木文创` 下拉锚点为 `good-cultural`，保证桌面端和移动端下拉一致。
- `custom/check-navigation-consistency.mjs`、`custom/check-release-readiness.mjs`：同步检查脚本对 `good-cultural` 的要求，避免脚本继续检查旧锚点。

## 7. 本轮优点

本轮优点是先恢复稳定基线，再做局部改动，避免继续在损坏 HTML 上修补。Header、移动端菜单和好物六类体系回到可验证状态，且 V1.18.6 的正确需求没有丢失。

检查脚本也被同步到新锚点口径，后续如果 `good-creative` 回潮或好物下拉缺项，会继续被导航一致性和发布预检拦截。

## 8. 本轮缺点与代价

由于文创锚点从 `good-creative` 统一改为 `good-cultural`，全站公开 HTML 的 Header 下拉链接都会产生差异，改动文件数比只修两页更多。这是为了保证全站导航一致，而不是只让首页通过检查。

本轮没有继续推进 V1.18.7 的文章质感精修；此前未提交的本地 V1.18.7 改动已被丢弃，避免混入本次紧急修复。

## 10. 本轮结论

本轮确认 V1.18.6 的主要问题来自批量替换误伤，不应继续使用整块 HTML 正则替换。对这类静态站点页面，后续需要优先恢复稳定文件，再进行可审查的小范围精确修改。

当前可继承的新结论：

- 文创分类锚点统一为 `good-cultural`。
- 六类体系仍为：柚木家具、柚木地板、柚木整装、柚木户外、柚木收藏、柚木文创。
- 好物卡片可以保留视觉块，但不显示 01-06 编号文本。
- `content-depth`、`release-readiness`、`navigation-consistency` 必须作为提交前硬门槛。
