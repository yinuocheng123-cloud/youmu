# 版本记录：v1.17-good-things-positioning-rebuild

## 1. 本轮目标

本轮目标是完成“柚喜饰界｜V1.17 柚木好物栏目定位重构版”，把前台“柚木好物”从偏应用场景、偏项目方案的页面，调整为“全球柚木生活方式精选库”。

本轮属于栏目定位与前台表达重构，但不改变 URL 目录结构：`solutions/` 路径继续保留，前台导航仍使用“柚木好物”。本轮不新增后台、登录、商城、支付或数据库，不修改 GitHub Actions workflow，也不修改 `forms/form.js`。

## 2. 背景与上下文

本轮承接 `4402437 V1.16.1 真实推荐厂商补充与可信度收口`。V1.16.1 的核心结论是推荐厂商资料必须坚持真实性边界，宁可少写，不可编造；厂商资料、好物内容和企业合作都不能写成认证、背书或交易承诺。

当前问题是 `solutions/index.html` 虽然前台叫“柚木好物”，但内容仍然围绕整装、地板、庭院、茶室和家具五个应用场景展开，阅读感更像方案页或顾问式判断页，不像用户用来发现、收藏和参考柚木生活方式内容的精选库。

## 3. 问题分析

“柚木知识”负责解决认知，“推荐厂商”负责解决选择，“社群交流”负责解决连接。如果“柚木好物”继续写成空间应用方案，就会和知识页、旧专题页、推荐厂商页互相重叠。

本轮真正要解决的不是多写内容，而是让栏目功能更清楚：柚木好物负责“发现”。它应该让用户看到柚木家具、地板、茶室、户外、收藏和文创等生活方式方向，而不是继续教用户如何判断整装方案或如何对接企业资料。

## 4. 候选方案比较

方案一是保留五个旧场景，只微调标题。优点是改动最小，缺点是无法改变用户对“好物=方案页”的感受，因此放弃。

方案二是新增全新的 `goods/` 目录。优点是语义更干净，缺点是会改变现有路径、导航和历史链接，不符合本轮“路径继续保留”的要求，因此放弃。

方案三是在 `solutions/` 路径内重构前台表达，将主入口改为六类生活方式精选库，旧专题页保留为兼容入口。该方案满足定位重构，也控制了链接风险，因此采用。

## 5. 最终决策

最终采用方案三：重构 `solutions/index.html` 的主内容，把页面改为“Teak Lifestyle Picks / 柚木好物”，并按六类组织：

- 柚木家具
- 柚木地板
- 柚木茶室
- 柚木户外
- 柚木收藏
- 柚木文创

首页同步改为发现型好物卡片；推荐厂商页补充“企业也是柚木好物内容来源”；企业合作页补充企业可提供产品图、空间图、案例资料、工艺说明、使用故事和好物推荐。

## 6. 具体实现

- `solutions/index.html`：删除五个应用场景式主内容，改为六类精选方向、近期可整理的好物样例和好物来源说明。
- `index.html`：首页柚木好物模块改为家具、茶室、户外、收藏四个发现型入口，不再展示整装/地板/庭院等方案卡。
- `data/site-content.js`：同步首页数据源中的柚木好物标题、副标题、卡片、标签和说明。
- `about/index.html`：轻量同步栏目说明，将“空间应用”口径调整为发现家具、地板、茶室、户外、收藏和文创内容。
- `vendors/index.html`：补充推荐厂商与柚木好物的关系，说明企业产品和空间资料可进入好物分类。
- `cooperation/index.html`：补充企业可提供内容类型，并说明这些内容可进入柚木知识、柚木好物和推荐厂商。
- `styles.css`：新增 V1.17 好物精选库样式，包括好物方向卡、样例网格、来源说明和移动端单列规则。
- `custom/check-release-readiness.mjs`：新增 `solutions/index.html` 的 V1.17 防回退检查。
- `custom/check-content-depth.mjs`：新增柚木好物六类分类与旧方案页表达检查。
- `custom/client-preview-note.md`、`custom/launch-checklist.md`、`custom/public-review-checklist.md`：补充 V1.17 说明与验收项。

## 7. 本轮优点

本轮让“柚木好物”的栏目职责更清楚：用户进入该栏目后，不再先看到空间判断和企业资料承接，而是先看到可发现、可收藏、可参考的柚木生活方式方向。

推荐厂商与企业合作的关系也更自然：企业不是被写成认证对象，而是可以提供产品图、空间图、工艺说明和使用故事，成为好物内容的来源之一。

检查脚本加入了防回退规则，后续如果把 `solutions/index.html` 写回 `Teak Project Gallery`、五个应用场景、好物方案、价格、库存或购买口径，会在本地预检中暴露。

## 8. 本轮缺点与代价

旧专题页 `solutions/whole-decoration.html`、`solutions/flooring.html`、`solutions/outdoor.html`、`solutions/tea-room.html`、`solutions/furniture.html` 本轮仍保留原路径和主体结构，只在主入口中下沉，不做完整合并。这是为了避免 404 和大范围改动，但也意味着旧专题页与新好物库之间仍存在历史过渡痕迹。

首页好物模块从原来的图文方案卡改为文字型精选方向卡，视觉上更克制，但图片展示感减少。后续如果有真实产品图、空间图或授权素材，可以再把好物库升级为更丰富的图文精选流。

## 10. 本轮结论

V1.17 的核心结论是：柚木好物应当承担“发现”功能，而不是继续承担知识解释、方案判断或厂商选择功能。栏目边界调整为：柚木知识解决认知，柚木好物解决发现，推荐厂商解决选择，社群交流解决连接。

本轮没有新增虚假商品、真实价格、库存、购买入口、平台认证、交易担保或企业合作事实，也没有新增后台、登录、商城、支付或数据库。

## 11. 下一轮建议

1. 如后续获得企业授权图片或公开来源，可优先为六类好物补充真实图文样例。
2. 旧专题页可单独安排一轮“空间专题 / 应用参考”收口，减少与新好物库之间的历史口径差异。
3. 若未来要做真实好物详情页，应先制定来源、图片授权、品牌事实和交易边界规则，避免把精选库误做成商城。
