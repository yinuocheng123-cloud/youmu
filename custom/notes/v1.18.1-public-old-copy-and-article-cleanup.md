# 版本记录：v1.18.1-public-old-copy-and-article-cleanup

## 1. 本轮目标

本轮目标是完成“V1.18.1 公网旧口径与文章完整性总清理”，重点解决公网仍可见的 V1.17 以前旧表达、柚木好物六类口径不统一、知识文章导语重复、文章底部内部说明和检查脚本拦截不足的问题。

本轮属于发布前内容清理与可维护性提升，不新增后台、登录、商城、支付、数据库，不改变首页结构，不新增购物车、报价展示、库存、确认入口或真实厂商背书。

## 2. 背景与上下文

本轮承接 `custom/notes/v1.18-good-things-archive-first-batch.md`。V1.18 已经新增 30 个 `solutions/goods/` 好物档案页，并把柚木好物主体系推进到六类：柚木家具、柚木地板、柚木整装、柚木户外、柚木收藏、柚木文创。

公网复核时发现，部分公开页面仍残留 V1.17 以前的旧表达，例如“庭院户外”“茶室会客”“家具好物”“Teak Project Gallery”“五个应用场景”等旧口径，也有知识文章仍带导语重复、相关内容段落过重和 Footer 内部话的问题。

## 3. 问题分析

表面问题是旧词残留，核心问题是多个历史阶段的页面模板与文案口径叠加后，公网内容还没有完全收口。尤其是知识文章中“导语 + 第一段”重复，会让文章像由模板拼接而来；“相关内容”段落如果写成阅读说明，也会让文章不像自然长文。

另外，旧检查脚本虽然已经覆盖部分旧词，但对文章导语重复、文章 Footer 内部审核话、Teak Project Gallery、五个应用场景和“真实资料仍需以后续人工核验为准”等回潮项拦截还不够集中。

## 4. 候选方案比较

方案一：只替换 `solutions/index.html` 中的旧词。

优点是改动最小；缺点是旧词仍会从首页、知识页、厂商页、表单页和候选资料页继续露出，无法满足“全站公开 HTML 搜索并清理”的要求，因此放弃。

方案二：全站机械替换所有旧词，不处理文章结构。

优点是效率高；缺点是知识文章仍会保留导语重复和说明书感，无法解决文章完整性问题，因此放弃。

方案三：全站旧词替换 + 重点文章重写 + 检查脚本升级。

优点是能同时解决公网口径和后续回潮拦截；缺点是改动面较方案一更大，需要更完整验证。本轮最终采用该方案。

## 5. 最终决策

最终采用“公开页面旧词清理、六篇重点知识文章自然化重写、检查脚本升级、记录归档”的方式。

好物体系继续沿用 V1.18 的六类，不新增栏目，不改部署 workflow，不处理真实厂商背书，不替换真实素材。

## 6. 具体实现

实际修改范围包括：

- `index.html`：清理首页知识入口中的旧词，避免“下单”“茶室会客”等旧表达继续出现在公网。
- `data/site-content.js`：同步首页数据源口径，避免运行时或后续维护重新输出旧表达。
- `solutions/index.html`：复核并清理好物页旧口径，继续保留六类体系。
- `solutions/goods/*.html`：统一可见文案中的交易化和旧说明口径。
- `solutions/guides/*.html`：清理底部“判断路径 / 下单”等旧说明。
- `knowledge/index.html`、`knowledge/*.html`、`knowledge/topics/*.html`：清理旧词，并重点重写六篇文章。
- `vendors/*.html`、`vendors/candidates/index.html`：清理“家具好物 / 茶室会客 / 平台认证 / 交易担保”等旧表达。
- `cooperation/index.html`、`about/index.html`、`articles/*.html`、`cases/*.html`、`forms/*.html`：同步清理公网可见旧词。
- `custom/check-release-readiness.mjs`：新增旧口径和高风险词硬拦截。
- `custom/check-content-depth.mjs`：新增文章导语重复、重点文章小节拼接、相关链接数量和文章 Footer 内部话检查。
- `custom/check-navigation-consistency.mjs`：强化全站柚木好物桌面端与移动端下拉检查。
- `custom/check-site-links.mjs`：保留全站链接与 sitemap 检查，并继续确认 sitemap 不包含 `custom/`。

## 7. 本轮优点

- 公网旧口径清理范围从单页扩大到公开 HTML 与数据源。
- 柚木好物全站继续统一为六类，降低后续维护回退风险。
- 六篇重点知识文章改成自然文章结构，不再像模板小节拼接。
- 检查脚本从“发现部分旧词”升级为“拦截重点回潮项”。

## 8. 本轮缺点与代价

- 本轮涉及公开页面数量较多，虽然多数是旧词替换，但仍需要依赖脚本验证降低遗漏风险。
- 部分历史页面文件名仍保留旧路径或英文 slug，这是为了避免旧链接失效，本轮只清理公网可见口径，不做 URL 迁移。
- 部分页面仍是内容整理页，不是最终真实厂商资料或真实案例页，真实资料仍需后续按授权和来源逐步完善。

## 9. 验证与结果

本轮已执行并通过以下命令：

- `node --check script.js`
- `node --check data\site-content.js`
- `node --check forms\form.js`
- `node custom\check-home-links.mjs`
- `node custom\check-site-links.mjs`
- `node custom\check-encoding.mjs`
- `node custom\check-content-depth.mjs`
- `node custom\check-release-readiness.mjs`
- `node custom\check-navigation-consistency.mjs`
- `node custom\check-clickable-topic-items.mjs`

本地验证结果：全部通过。公开旧词扫描未再发现 Teak Project Gallery、五个应用场景、庭院户外、茶室会客、家具好物、好物方案、人工核验、阅读路径、继续看、下一步可以看、平台认证、官方推荐、交易担保、价格、库存、下单、立即购买等旧口径或高风险表达。

## 10. 本轮结论

V1.18 的六类体系需要靠公开页面、数据源和检查脚本同时守住。只改页面不足以防止回潮，必须把“Teak Project Gallery”“五个应用场景”“庭院户外”“茶室会客”“家具好物”和文章内部审核话写入检查规则。

知识文章应当像完整文章，而不是导语、小节、阅读提示和内部说明拼起来的页面。底部边界说明保留为一句自然提示即可。

## 11. 下一轮建议

1. 继续为 V1.18 第一批 30 个好物档案页补更稳定的公开资料来源链接。
2. 做 V1.18 第二批好物档案时，沿用本轮检查脚本，先跑检查再进入提交。
3. 若后续要处理候选厂商资料库，可以再单独做一轮“公开资料来源与可展示边界”整理。
