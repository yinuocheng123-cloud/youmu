# 版本记录：V1.16 正式上线冲刺版

## 1. 本轮目标

本轮目标是完成“柚喜饰界｜V1.16 正式上线冲刺版”。重点不是继续扩技术功能，而是把网站从“已经能看”推进到“可以正式推荐给企业和用户访问”的状态。

本轮优先处理三类问题：第一，全站公开页面中的建设期痕迹、后台口吻和资料说明口吻；第二，推荐厂商从方向导览进一步升级为品牌馆入口；第三，新增企业合作页，给企业一个正式了解展示方式的入口。

本轮明确不新增后台、登录、商城、支付、数据库，不修改 GitHub Actions workflow，不改变现有栏目路径结构。

## 2. 背景与上下文

本轮承接 `821ec0b V1.15.4 正式上线前前台口径精修`。V1.15.4 已经清理了大量建设期语言、内部说明语言和未完成口径，但 V1.16 进一步要求公开前台不再像预览站，而是像可以正式对外推荐访问的内容站。

V1.15.3G 已完成短标题、CTA、模块头部和移动端节奏的版式收口；V1.15.4 已完成前台口径精修。因此本轮不再重做设计系统，而是在既有视觉和导航结构上做上线冲刺：清理全站残留词、强化首页主次、补齐企业合作入口，并新增四个推荐厂商资料页。

## 3. 问题分析

本轮的核心问题不是单个页面打不开，而是公开前台仍可能让用户感觉网站处于搭建过程。例如“样板”“示例”“待确认”“会员站”“人工确认”“审核”等词，即使技术上不影响访问，也会降低正式感和可信度。

推荐厂商页在 V1.15.4 后已经能阅读，但整体仍偏“按方向查看企业资料”，真实品牌感不足。用户要求新增四个推荐厂商资料页，因此推荐厂商首页需要从方向导览升级为品牌馆入口，同时保留公开资料观察库边界。

企业合作入口此前主要由厂商申请页承接，容易一打开就像表单系统。V1.16 要求新增独立企业合作页，因此本轮把合作说明前置，把原有申请页下沉为资料摘要整理页。

## 4. 候选方案比较

方案一是只做关键词替换，不新增页面。这种方案改动小，但不能满足新增四个厂商资料页和企业合作页的要求，也无法解决推荐厂商页品牌馆感不足的问题，因此放弃。

方案二是大规模重写知识体系、案例体系和全部厂商页面。这能更彻底，但风险过高，容易推翻 V1.15.3F、V1.15.3G 和 V1.15.4 已稳定的内容与样式，也不符合“上线冲刺”的节奏，因此放弃。

方案三是“全站残留词清理 + 首页主次调整 + 推荐厂商品牌馆 + 新增企业合作页 + 新增四个厂商资料页 + 检查脚本补强”。该方案覆盖用户明确要求，同时不新增系统功能，是本轮采用方案。

## 5. 最终决策

最终采用方案三。具体决策如下：

- 将首页模块顺序调整为认识柚喜、柚木知识、推荐厂商、柚木好物、社群交流。
- 推荐厂商首页新增四个厂商资料卡，强化品牌馆入口感。
- 新增 `cooperation/index.html`，作为企业合作正式说明页。
- 新增四个推荐厂商资料页：臻藏柚木生活、悦百家柚木地板、雪莲花柚木家具、壹信柚木。
- 原 `forms/vendor-apply.html` 保留表单摘要功能，但改为企业资料整理承接页。
- 公共页面统一清理“样板、示例、演示、占位、待补、待确认、建设中、会员站、人工确认、审核”等词。
- 发布预检脚本补充 V1.16 风险词，防止后续回退。

## 7. 本轮优点

本轮让首页主次更清楚：用户先认识柚喜，再看柚木知识，然后看到推荐厂商，最后进入柚木好物和社群交流。推荐厂商页从方向说明变成品牌馆入口，公开访问时更像正式栏目。

企业合作页把合作说明前置，原表单页不再承担第一屏合作解释，降低了“后台申请系统”的观感。

发布预检加入 V1.16 风险词后，后续如果又写回“样板、示例、待确认、会员站、人工确认”等词，会在本地检查中被拦截。

## 8. 本轮缺点与代价

本轮为了全站清理同类词，对多个公开页面做了小幅批量替换，diff 覆盖范围较大。虽然每处都是轻量文案收口，但后续仍建议公网逐页抽查，确认没有个别句子因批量替换显得生硬。

四个新增厂商资料页因未检索到稳定可引用的官网、公众号或公开品牌介绍，本轮没有写入具体历史、案例、认证、联系方式或规模信息。这样做避免了编造，但页面内容相对克制，仍需要企业后续提供可公开材料后再丰富。

旧文件名中仍保留 `sample` 这类英文路径名，主要为了避免大规模改 URL 和 sitemap。前台可见文字已改为“空间参考”，但 URL 层面的历史命名未在本轮重命名。

## 10. 本轮结论

V1.16 的核心结论是：上线前最重要的是让用户只看到内容本身，而不是看到建设过程、后台流程或资料整理痕迹。推荐厂商需要有品牌馆感，企业合作需要有正式说明页，案例应表达为空间参考而不是伪装成真实项目。

资料真实性处理上，本轮坚持不编造厂商历史、案例、认证、联系方式或规模。四个新增厂商页只按公开资料整理口径展示可关注方向，并把风险边界压缩到底部说明。

本轮没有新增后台、登录、商城、支付、数据库，没有修改 GitHub Actions workflow，也没有修改 `forms/form.js` 核心摘要逻辑。

## 11. 下一轮建议

1. 企业真实资料到位后，优先补充四个新增厂商页的官网或公众号来源、授权图片、服务区域、产品资料和案例材料。
2. 公网上线后人工抽查首页、推荐厂商、四个厂商资料页、企业合作页、观察库、厂商申请页和案例页，确认没有建设期口吻回潮。
3. 如果继续推进正式上线，可单独做一次 URL 命名整理，把历史 `sample` 路径迁移为更自然的 `reference` 或 `inspiration`，但需要同步全站链接和 sitemap。
