# 版本记录：v1.17.2-good-things-secondary-system-rebuild

## 1. 本轮目标

本轮目标是完成“柚喜饰界｜V1.17.2 柚木好物二级体系重构版”，把 `柚木好物` 的二级体系统一收口为：

- 柚木家具
- 柚木地板
- 柚木整装
- 柚木户外
- 柚木收藏
- 柚木文创

同时取消把“柚木茶室”作为六大主分类之一的做法，并把相关内容并入“柚木整装 / 柚木家具 / 柚木文创”的交叉体系中。

本轮重点不是继续扩充内容数量，而是修正栏目逻辑、分类口径和旧专题页定位。

## 2. 背景与上下文

本轮直接承接以下记录：

- `custom/notes/v1.17.1-good-things-content-flow.md`
- `custom/notes/v1.17-good-things-positioning-rebuild.md`
- `custom/notes/v1.16.1-real-vendor-source-and-launch-trust-polish.md`

上一轮 V1.17.1 已经把 `solutions/index.html` 从分类说明页推进成 72 条内容流，但当时的六类仍是：

- 柚木家具
- 柚木地板
- 柚木茶室
- 柚木户外
- 柚木收藏
- 柚木文创

这套结构解决了“内容不够多”的问题，但没有完全解决“二级体系不够稳定”的问题。`柚木茶室` 仍然既像主分类，又像空间专题，与 `solutions/whole-decoration.html`、旧整装逻辑和茶桌 / 茶柜 / 地台 / 墙板等内容之间存在边界模糊。

本轮继续沿用 V1.17 和 V1.17.1 的核心边界：

1. 前台栏目名继续保留“柚木好物”
2. `solutions/` 路径继续保留，不做 URL 大改
3. 不新增后台、登录、商城、支付、数据库
4. 不改 GitHub Actions workflow
5. 不改 `forms/form.js`
6. 不新增真实商品、价格、库存、购买链接
7. 不编造真实案例、真实厂商、真实认证或交易承诺

## 3. 问题分析

当前问题不是栏目定位错误，而是二级体系还保留了过渡痕迹。

表面现象是：

- 主入口已经是内容流，但第三大类仍然叫“柚木茶室”
- 旧 `tea-room.html` 仍容易被理解成与整装、地板、户外、家具平级的主入口
- 首页、推荐厂商页、企业合作页和旧专题页中的二级名称还夹杂“庭院户外”“家具好物”“茶室会客”等历史说法

真正的核心矛盾是：

1. “茶室”本质上更像交叉空间主题，而不是长期稳定的一级二级分类
2. “整装”与“茶室”之间的边界在旧体系中被拆散了
3. 旧专题页不能删除，但如果不重新定义，就会持续把主入口拉回旧方案逻辑

因此本轮的重点不是增加条目，而是让分类更长期、搜索更清楚、专题与主类关系更稳定。

## 4. 候选方案比较

### 方案一：保留 V1.17.1 六类不动，只改首页文字

优点：

- 改动最小
- 风险最低

缺点：

- 没有解决“茶室是不是主类”的结构问题
- 旧专题页和主入口边界仍然模糊

放弃原因：只能修词，不能修体系。

### 方案二：直接删除 `tea-room.html`，完全并入整装

优点：

- 结构最干净
- 分类更纯粹

缺点：

- 直接造成旧链接风险
- 不符合“旧页面不要删除，避免 404”的要求

放弃原因：风险过高，不符合本轮边界。

### 方案三：主入口改成六类新体系，`tea-room.html` 降级为交叉专题

核心思路：

- `solutions/index.html` 主入口六类改成“柚木整装”
- `tea-room.html` 保留原路径，但明确改成“柚木茶室空间”专题
- 72 条内容保留总量不变，只对第三分区做重归类
- 首页、推荐厂商、企业合作和检查脚本同步到新体系

优点：

- 不改 URL
- 不丢旧页面
- 主入口逻辑更清楚
- 能继续承接旧内容和未来扩展

缺点：

- 需要同时修改主入口、旧专题、首页、厂商页和脚本
- 需要处理大量旧文案回流

最终选择原因：这是在当前边界下最稳、最完整的重构方式。

## 5. 最终决策

本轮最终决策如下：

1. `solutions/index.html` 的六大主分类统一为：
   - 柚木家具
   - 柚木地板
   - 柚木整装
   - 柚木户外
   - 柚木收藏
   - 柚木文创
2. 原“柚木茶室”主分区替换为“柚木整装”分区，id 从 `good-tea-room` 改为 `good-whole-decoration`
3. 原茶室内容不删除，而是重新吸收到：
   - 茶桌等偏单品内容：继续留在柚木家具
   - 墙板、地台、茶室空间等内容：并入柚木整装
   - 小器物与日常小件：延续到柚木文创
4. `solutions/tea-room.html` 改为“柚木茶室空间”专题页，不再作为六大主类入口
5. 公开页面不再把“精选”作为二级栏目名使用

## 7. 本轮优点

本轮的主要优点是：

- 柚木好物的二级体系终于和旧专题页逻辑拉开边界
- “柚木整装”成为长期更稳定的分类承接
- “柚木茶室”不再以主类身份干扰结构，但内容没有丢
- 旧链接全部保留，没有制造 404
- 首页、推荐厂商、企业合作、主入口和旧专题页口径同步
- 检查脚本可以更早拦截“精选 / 茶室主类 / 旧口径”回流

## 8. 本轮缺点与代价

本轮也有明确代价：

- `tea-room.html` 仍然存在，体系上比完全删除更复杂，但这是为兼容历史链接做的必要妥协
- 整装分区虽然完成替换，但很多条目仍是方向卡片，不是完整图文专题
- 旧 `solutions/guides/` 中的若干长文仍保留历史专题结构，只做了名称和 breadcrumb 收口，没有全面重写内容逻辑
- “精选”虽然已从当前主入口和二级体系中移除，但历史版本记录与旧说明文件仍保留历史表述，这部分作为历史档案不做回写

## 10. 本轮结论

本轮最重要的结论是：

“柚木茶室”更适合作为交叉专题，而不是长期稳定的主分类；真正稳定的六类体系应当是“柚木家具、柚木地板、柚木整装、柚木户外、柚木收藏、柚木文创”。

同时被再次验证的结论还有：

1. 旧路径必须保留，不能用删除页面来解决结构问题
2. 主页、厂商页、合作页和检查脚本如果不同步，主入口再怎么改也会被旧口径拖回去
3. 与其放宽预检脚本，不如直接把前台文案改得更自然、更用户向
