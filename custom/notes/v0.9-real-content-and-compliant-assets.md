# 版本记录：v0.9-real-content-and-compliant-assets

## 1. 本轮目标

本轮目标是在 V0.8 甲方预览确认包整理完成、首页方向和 Logo 方向已确认的基础上，把首页从“示例占位感较强”的预览版推进到“内容更丰满、图片更真实、来源可追溯、适合继续给甲方预览”的状态。

本轮目标属于内容补充、合规素材替换和可维护性提升，不新增后台、登录、商城、支付、数据库，也不处理真实厂商背书、真实资质认证或真实案例真实性。

## 2. 背景与上下文

本轮承接 `custom/notes/v0.8-client-review-package.md`。V0.8 已经完成甲方预览确认包，明确首页定位为“柚木知识平台＋好物推荐＋推荐厂商展示＋企业微信社群承接入口”，并强调当前推荐厂商、好物、图片和联系方式仍需后续替换。

修改前首页已经具备完整结构：Hero、三大价值、柚木知识、柚木好物、推荐厂商、推荐标准、企业微信社群和 Footer。V0.7 已经完成喜字方向 Logo 接入，V0.8 已经整理 Logo 和首页说明。本轮不改变结构，只替换图片资源、深化文案、补足素材来源记录。

任务开始时按项目规则尝试执行 `git pull --ff-only`，但当前目录不是 Git 仓库，命令返回 `fatal: not a git repository`。因此本轮没有初始化 Git，也没有 commit、push、deploy。

## 3. 问题分析

当前首页的核心问题不是结构缺失，而是预览素材仍以 SVG 占位为主，文案密度偏轻，容易让甲方感觉网站仍停留在早期 demo 阶段。

真正的核心矛盾是：既要让页面看起来更接近真实上线前预览，又不能冒险使用版权不清、来源不明或真实厂商属性不明的图片，也不能把样板位写成真实厂商案例。

主要风险包括：

- 图片版权风险：不能使用百度图片、电商图、社交平台用户图、品牌官网图或摄影师作品站授权不明图片。
- 厂商误导风险：推荐厂商区域不能让用户误以为当前图文对应真实合作厂商。
- 商城化风险：好物栏目不能出现价格、购物车、立即购买或商品详情页表达。
- 资料真实性风险：本轮不能声称真实认证、真实案例、行业背书或真实厂商合作关系。

## 4. 候选方案比较

### 方案一：继续使用已有 SVG 占位图

核心思路：不引入外部图片，只深化文案。

优点：完全无版权风险，改动最小。

缺点：首页真实感不足，无法完成本轮“图片更真实、适合继续预览”的目标。

结论：放弃。该方案安全但预览效果提升有限。

### 方案二：使用公共图库开放授权图片

核心思路：从明确开放授权图库中筛选适合柚木生活方式、木作空间、工坊和材质细节方向的图片，下载到本地并建立来源台账。

优点：视觉真实度明显提升；图片来源和授权可追溯；不会依赖远程图片；后续可替换为甲方真实图。

缺点：图库图片仍不是甲方真实素材，部分图片只能表达氛围方向，不能作为真实案例或真实厂商证明。

结论：采用。该方案在合规、安全和预览效果之间更平衡。

### 方案三：使用自制抽象 SVG 或纹理图

核心思路：用自制图形模拟木纹和空间氛围。

优点：版权可控，体积轻。

缺点：真实感和生活方式表达弱，无法充分体现甲方希望看到的真实内容填充效果。

结论：未采用为主方案，仅保留为未来素材不足时的备选。

## 5. 最终决策

最终采用“开放授权图库素材＋本地下载＋来源台账＋文案合规标注”的方案。

关键决策：

- 图片统一下载到 `assets/images/`，不引用远程图片。
- 只使用 Pexels 等明确开放授权图库素材，并在 `custom/image-source-log.md` 记录来源、授权和用途。
- 厂商区继续保持“样板位”和“展示方向”口径，不写真实厂商故事或真实案例。
- 好物区只讲适合场景、判断要点和维护关注点，不加入价格、购物车或立即购买。
- 企业微信承接路径不变，所有主要 CTA 继续指向二维码承接区。

## 7. 本轮优点

- 页面真实感提升：首屏、栏目卡片、好物区、厂商区和企业微信承接区都从 SVG 占位转为更接近真实网页预览的图片。
- 合规可追溯：每张图都在 `custom/image-source-log.md` 中记录来源页面、平台、授权说明和使用位置。
- 文案更成熟：知识、好物、厂商推荐标准和社群承接文案都补充了判断逻辑，页面不再只是概念口号。
- 风险边界更清楚：厂商区明确为样板位，不代表真实推荐厂商、真实案例或合作关系。
- 结构保持稳定：没有重做首页结构，没有改变导航顺序，没有新增商城、后台或数据库。
- 本地资源更可靠：图片全部放在 `assets/images/`，页面不依赖远程图片或远程字体。

## 8. 本轮缺点与代价

当前方案最大的代价是：图片虽然具有开放授权记录，但仍然不是甲方真实图片，也不是柚木厂商真实案例。它们适合继续预览视觉气质和内容密度，不适合直接当作最终上线素材。

另一个代价是素材来自公共图库，个别图片只能表达“木作、庭院、工坊、材质”的氛围，不一定严格等同于柚木。为了避免误导，页面和记录中均使用“木质”“木作”“样板图”“展示方向”等更稳妥措辞。

推荐厂商区域仍未完成真实资料替换。本轮刻意不核验厂商真实性、不补真实背书、不编造案例，因此后续上线前仍必须做真实资料、授权和公开发布确认。

## 10. 本轮结论

V0.9 已经把首页从“方向确认包”推进到“内容和图片更丰满的继续预览版”。当前版本适合让甲方进一步感受真实内容密度、开放授权图片替换后的视觉气质，以及企业微信承接路径。

本轮确认的经验是：在真实资料未最终到位前，可以使用开放授权图库素材提升预览质量，但必须把素材状态写清楚，不能把氛围图包装成真实厂商、真实案例或项目背书。

## 11. 下一轮建议

1. 优先做本地截图和视觉复核，确认桌面端、手机端图片裁切、二维码区和 CTA 滚动正常。
2. 甲方提供真实素材后，按 `custom/image-source-log.md` 的格式逐张替换并补齐授权记录。
3. 推荐厂商资料进入真实替换阶段前，先确认厂商是否允许官网展示、是否允许平台分发、是否有不可公开客户或案例信息。
