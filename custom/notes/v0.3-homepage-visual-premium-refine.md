# 版本记录：v0.3-homepage-visual-premium-refine

## 1. 本轮修改目标
本轮目标是在 V0.2 首页结构和文案已经成立的基础上，进行高级视觉精修。

本轮不新增后台、登录、商城、支付或数据库，不重做整站，只聚焦减少 CSS 条纹占位感，建立更接近真实高端柚木生活方式官网的图片承载结构，并强化企业微信二维码承接。

## 2. 实际修改文件
- `index.html`：将 Hero、知识、好物、推荐厂商等图片区域改为明确的 `<img>` 结构；接入真实企业微信二维码；补充真实图片和二维码替换注释；推荐厂商卡片增加“柚喜饰界内容共创展示”小标签；第三张厂商卡按钮改为“了解产品工艺”。
- `styles.css`：移除 Hero 和卡片区域明显条纹/色块占位，改为图片裁切、柔和遮罩、卡片阴影、标准卡片与二维码视觉样式；优化移动端二维码和图片比例。
- `assets/`：新增统一风格的 SVG 视觉资产，并复制真实企业微信二维码。
- `custom/notes/v0.3-homepage-visual-premium-refine.md`：新增本轮记录。

## 4. 哪些 CSS 占位图被替换
本轮替换了以下 V0.2 中的 CSS 占位视觉：

- Hero 原来的渐变、重复线条和右侧矩形色块占位，替换为 `assets/hero-teak-lifestyle.svg` 图片承载结构，并保留深色文字遮罩。
- 柚木知识区四张卡片原来的 CSS 木纹/条纹背景，替换为四张 SVG 图片。
- 柚木好物区三张卡片原来的 CSS 条纹背景，替换为三张 SVG 图片。
- 推荐厂商区三张卡片原来的 CSS 条纹背景，替换为工坊、展厅、木作细节三张 SVG 图片。
- Footer 公众号二维码占位从 CSS 粗略块状图改为 `assets/qr-placeholder.svg`。

## 5. 企业微信二维码位置和替换方式
企业微信二维码位置在 `index.html` 的 `.qr-panel` 内。

当前已使用真实二维码：

```html
<img class="qr-code-image" src="./assets/wecom-qr.jpg" alt="扫码添加柚喜顾问企业微信二维码" />
```

后续如果要替换为新的企业微信二维码，可以替换 `assets/wecom-qr.jpg`，或按页面注释新增 `assets/wecom-qr.png` 后调整 `src`。

## 6. 真实图片后续替换方式
所有视觉图都已经改成 `<img>` 或明确图片资源结构。后续替换真实图片时，只需要替换对应 `src`：

- Hero：`assets/hero-teak-lifestyle.svg`
- 知识区：`assets/knowledge-*.svg`
- 好物区：`assets/product-*.svg`
- 推荐厂商区：`assets/vendor-*.svg`

替换为真实 JPG/PNG/WebP 时，保留现有 class 即可复用裁切、遮罩和响应式样式。
