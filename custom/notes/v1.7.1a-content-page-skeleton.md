# 版本记录：v1.7.1a-content-page-skeleton

## 1. 本轮目标

本轮目标是单独完成“柚喜饰界｜V1.7.1A 内容页骨架生成版”，在不延续上一轮长指令、不写完整长文的前提下，为首页已有知识、方案、厂商等入口补齐最小静态内容页。

本轮属于功能补充与信息架构骨架建设，重点是页面存在、链接闭环、sitemap 初步更新和来源记录框架建立。

## 2. 背景与上下文

本轮承接：

- `custom/notes/v1.6-mobile-nav-fix.md`
- `custom/notes/v1.5-pages-workflow-cleanup.md`
- `custom/notes/v1.0-knowledge-products-and-nav.md`

V1.6 的核心结论是移动端 Header、抽屉菜单层级和点击顺序已经修复，本轮不能破坏移动端导航。V1.5 的核心结论是 GitHub Pages 只应发布公开运行所需文件，本轮不修改 workflow。V1.0 已经把柚木知识、好物方案和推荐厂商作为首页入口，但尚未形成独立内容页。

仓库根目录未发现实际 `AGENTS.md` 文件，本轮以用户消息中提供的 AGENTS 指令为准执行。

## 3. 问题分析

当前首页已有内容入口，但多数仍停留在首页锚点或企业微信承接区。用户进入知识、方案、厂商、案例、文章等方向时，没有独立页面可浏览，也没有 sitemap 收录路径。

本轮核心矛盾是：需要快速建立内容页结构，但不能把骨架误写成真实厂商、真实案例、真实认证、真实资质或真实交易能力。因此内容必须短、保守、可继续扩写，并在厂商与案例样板中明确真实性边界。

## 4. 候选方案比较

### 方案一：一次性扩写完整长文章

优点是内容看起来更完整。缺点是本轮明确要求不要写长文章，且未联网核实 FSC、PEFC、CITES、IUCN、木材数据库等来源，贸然扩写会增加资料真实性风险。

本轮放弃该方案。

### 方案二：只新增目录，不新增详情页

优点是改动更小。缺点是无法满足本轮指定页面清单，也无法形成首页主要卡片与按钮的链接闭环。

本轮放弃该方案。

### 方案三：新增全量指定页面，但每页只做短骨架

优点是满足页面清单、链接闭环和 sitemap 更新要求，同时保留后续扩写空间。缺点是内容深度有限，仍需要下一轮补资料来源与人工复核。

本轮采用该方案。

## 5. 最终决策

最终采用“全量指定页面 + 短内容骨架 + 明确真实性边界”的方案。

关键决策：

- 新增 `knowledge/`、`solutions/`、`vendors/`、`cases/`、`articles/`、`forms/`、`about/` 七个公开目录。
- 每个页面使用统一内容页 Header、Footer、面包屑、H1、导语、正文、相关链接、返回首页和咨询入口。
- 厂商样板页明确写入“不代表已有真实厂商入驻，不构成平台背书或交易担保”。
- 案例样板页明确写入“非真实成交案例，真实案例上线前需取得授权”。
- 首页主要入口从锚点或按钮补充为静态页面链接。
- 更新 `sitemap.xml`，只收录公开页面，不收录 `custom/`。
- 新增 `custom/content-source-log.md`，只建立来源记录框架，不伪造来源。

## 7. 本轮优点

- 首页主要入口具备独立页面承接，不再只有首页锚点。
- 内容页视觉与首页保持同一配色、圆角、按钮和木质调性。
- 厂商和案例页明确保留真实性边界，降低误导风险。
- sitemap 初步覆盖新增公开页面，便于后续发布。
- 来源记录框架已建立，后续补资料时有记录入口。
- 未新增后台、登录、支付、数据库或复杂表单逻辑。

## 8. 本轮缺点与代价

本轮页面仍是骨架，内容深度有限，不适合作为正式知识库直接对外宣传。

当前没有联网核实 FSC、PEFC、CITES、IUCN、木材数据库或公开教育资料，因此页面中不写具体认证结论、保护状态结论或权威背书。

表单页只是静态骨架，不保存数据、不发送消息、不产生审核流程。真实上线前需要补隐私说明、数据处理方式和人工响应流程。

首页链接检查脚本只检查首页本地页面链接存在性，未扩展为全站深度爬取脚本；这是为了遵守本轮“不新增复杂检查脚本”的边界。

## 10. 本轮结论

V1.7.1A 已将首页既有信息架构推进到“可访问的公开内容页骨架”阶段。当前最重要结论是：可以先建立页面路径和内容组织方式，但所有真实厂商、真实案例、认证资质和来源材料必须在后续版本中补充并人工确认。

本轮延续 V1.6 的移动端导航修复，不触碰 GitHub Actions，不引入远程图片、远程字体或远程脚本。

## 11. 下一轮建议

1. 优先补充 `custom/content-source-log.md` 中的 FSC、PEFC、CITES、IUCN、木材数据库和公开教育资料来源，并为文章页建立可核实引用。

3. 若准备接入真实厂商或案例，先建立资料授权清单与人工审核流程，再替换样板内容。
