# 版本记录：v1.18.8-good-things-anchor-navigation-fix

## 1. 本轮目标

本轮目标是修复柚木好物六类锚点与导航结构中的不一致问题。当前检查脚本已经要求文创分类统一使用 `good-cultural`，但公开页面仍有部分位置使用 `good-creative`，导致全站链接检查、发布预检和导航一致性检查存在失败风险。

本轮只修复结构问题，不改 30 个好物文章正文、不改知识文章正文、不改首页文案、不改推荐厂商页文案、不改企业合作页文案、不改 `forms/form.js`、不改 workflow，也不新增后台、登录、商城、支付或数据库。

## 2. 背景与上下文

本轮开始前执行了 `git status`、`git pull --ff-only` 和 `git log --oneline -5`。当前最新提交为 `cd5bb33 V1.18.7 好物剩余文章精修 + V1.18.6 首页误伤修复`，工作区干净且远程已是最新。

历史记录中最新可读的正式结构修复记录为 `custom/notes/v1.18.6-fix-good-things-nav-and-index-restore.md`。该记录已经提出文创分类锚点应统一为 `good-cultural`，但当前公开文件中仍残留 `good-creative`。

## 3. 问题分析

问题表面是链接目标不存在：全站导航和数据源中的柚木文创入口指向旧锚点 `good-creative`，而检查脚本和本轮要求都以 `good-cultural` 为准。

核心矛盾是同一个分类存在两个锚点口径。对于静态站点，这会让页面看起来仍能打开，但锚点滚动、全站链接检查和导航一致性检查不再稳定。

## 4. 候选方案比较

方案一：把检查脚本改回 `good-creative`。优点是改动少，缺点是违背本轮明确要求，也会继续保留多个锚点版本的风险。

方案二：只在 `solutions/index.html` 增加一个兼容锚点。优点是能让旧链接不报错，缺点是会继续留下 `good-creative` 与 `good-cultural` 并存，后续更难维护。

方案三：统一公开导航、好物页分类筛选、目标区块 id 和数据源为 `good-cultural`。优点是口径唯一，缺点是需要改 3 个文件中的多个锚点位置。最终选择方案三。

## 5. 最终决策

本轮采用精确补丁修改，不使用全局正则替换整块 HTML。

统一口径为：

- `id="good-cultural"`
- `./solutions/index.html#good-cultural`
- `../solutions/index.html#good-cultural`
- `#good-cultural`

并确认不再出现 `good-creative`、`good-culture`、`good-cultural-creative` 或 `good-wenchuang`。

## 6. 具体实现

实际修改文件：

- `index.html`：将首页桌面端和移动端“柚木好物”下拉中的柚木文创链接改为 `./solutions/index.html#good-cultural`。
- `solutions/index.html`：将好物页桌面端和移动端下拉、页面分类筛选链接改为 `good-cultural`，并将柚木文创区块 id 从 `good-creative` 改为 `good-cultural`。
- `data/site-content.js`：将数据源中的柚木文创入口改为 `./solutions/index.html#good-cultural`。

未修改文章正文、未修改 `forms/form.js`、未修改 workflow。

## 7. 本轮优点

本轮改动范围非常小，只修复锚点和导航结构。相比保留兼容锚点，统一为单一 `good-cultural` 口径更利于后续检查脚本和人工维护。

## 8. 本轮缺点与代价

本轮没有处理 V1.18.7 记录缺失的问题，也没有继续做内容精修。由于目标是结构修复，本轮不扩展范围。
