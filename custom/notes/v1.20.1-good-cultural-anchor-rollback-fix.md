# 版本记录：v1.20.1-good-cultural-anchor-rollback-fix

## 1. 本轮目标

本轮目标是复核并修复 `good-cultural` 锚点可能被旧恢复命令回退的问题，确保柚木文创分类只使用 `good-cultural`，不再出现 `good-creative`、`good-culture`、`good-wenchuang` 或 `good-cultural-creative`。

本轮属于结构回归复核与发布前阻塞排查，不改文章正文，不改 `forms/form.js`，不改 workflow，不改 sitemap 结构，也不新增后台、登录、商城、支付或数据库。

## 2. 背景与上下文

本轮承接 `custom/notes/v1.20-good-things-system-unification.md` 和 `custom/notes/v1.18.8-good-things-anchor-navigation-fix.md`。

V1.18.8 已经把柚木文创锚点统一为 `good-cultural`。本轮特别强调不再执行旧命令：

`git checkout 4f204fc -- index.html solutions/index.html data/site-content.js styles.css`

因为该命令会把已修复的 `good-cultural` 路径回退为旧状态。

## 3. 问题分析

本轮开始时执行 `git status`、`git pull --ff-only` 和 `git log --oneline -8`，确认当前 HEAD 为 `1a19f57 V1.20 柚木好物内容体系统一收口`，工作区干净且远端已是最新。

随后检查 `index.html`、`solutions/index.html`、`data/site-content.js` 和相关检查脚本，确认当前源码中 `good-cultural` 已存在且没有旧锚点别名残留。因此本轮没有对业务页面做代码修复，只做验证记录与完整检查。

## 4. 候选方案比较

方案一：重新从旧提交恢复页面文件再修复。

该方案被明确放弃，因为它正是上次造成锚点回退的风险来源。

方案二：在当前最新代码上只做精确搜索和必要修复。

这是本轮采用的方案。实际检查结果显示当前最新代码已经保持正确，因此无需改动页面正文或导航结构。

## 5. 最终决策

本轮不执行旧恢复命令，不修改文章正文，不修改好物页内容。只确认以下事实：

- `solutions/index.html` 存在 `id="good-cultural"`。
- 首页桌面端和移动端好物下拉指向 `./solutions/index.html#good-cultural`。
- 好物页自身桌面端和移动端好物下拉指向 `../solutions/index.html#good-cultural`。
- 页面内部六类入口使用 `#good-cultural`。
- `data/site-content.js` 使用 `./solutions/index.html#good-cultural`。
- 检查脚本继续以 `good-cultural` 作为唯一文创锚点。
