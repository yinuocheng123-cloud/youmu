# 版本记录：v0.1-homepage-demo

## 1. 本轮目标
本轮目标是完成“柚喜饰界｜柚木爱好者乐园”网站首页设计 demo，让页面可以直接本地预览，并作为后续继续开发的首页基线。

本轮目标类型属于功能补充与首轮视觉基线建立。它值得优先处理，是因为当前目录未发现既有项目文件，必须先建立一个能向客户确认方向的首页原型。

## 2. 背景与上下文
本轮开始时，`D:\ceshi\youxi` 目录为空，未发现 `.git`、`AGENTS.md`、`custom/notes/` 或任何业务代码文件。按用户提供的仓库规则尝试执行 `git pull --ff-only`，但当前目录不是 Git 仓库，因此无法拉取远程最新代码。

由于没有历史正式记录，本轮作为首轮方案建立基线。用户提供了一张高端柚木生活方式首页参考图，要求页面强调“柚木知识平台＋可信商家展示平台＋企业微信社群承接入口”，降低电商感，不出现价格、购物车、购买按钮。

## 3. 问题分析
表面问题是需要实现一个高端官网首页 demo。核心矛盾是：页面要有完整商业转化能力，但不能像普通家具商城，必须通过内容、标准、案例和社群承接来建立信任。

本轮主要约束如下：

- 当前项目为空，无法沿用现有技术栈。
- 不能依赖不可访问的远程图片。
- 不新增后台、数据库、登录、商城或支付。
- 所有按钮都需要滚动到企业微信二维码承接区。
- 移动端必须可读、可点、模块重排合理。

## 4. 候选方案比较
方案一：建立静态 HTML/CSS/JS 首页。

核心思路是用 `index.html`、`styles.css`、`script.js` 完成页面结构、视觉样式和滚动交互。优点是依赖最少、启动简单、可直接预览，适合当前空目录状态。缺点是暂时没有组件化能力，后续若接入 React/Next 需要迁移结构。风险较低。

方案二：新建 React 或 Next 项目。

核心思路是直接初始化现代前端项目，便于后续组件化开发。优点是扩展能力强。缺点是需要新增依赖、构建配置和安装流程，在当前用户明确要求不要随意重构或新增复杂工程的前提下成本偏高，也会增加验证风险。

最终选择方案一，因为当前目录为空且任务要求是可运行、可预览、可继续开发的首页 demo，静态方案最保守、最稳定。

## 5. 最终决策
最终采用静态单页首页方案。

关键设计决策：

- 用 CSS 渐变、重复纹理和抽象木作形状模拟柚木庭院生活视觉，避免依赖远程图片。
- 首页内容按照用户指定九个部分完整实现。
- 所有 CTA 统一绑定到 `#wechat`，突出企业微信社群承接。
- 产品区只表达“适合场景”和“了解判断”，不出现价格、购物车、立即购买。
- 商家区用“厂商故事”“真实案例”等表达方式，避免商家黄页感。
- 信任机制区以六个标准模块建立平台可信度。

## 6. 具体实现
本轮新增文件：

- `index.html`：首页结构，包含导航、Hero、三大价值、柚木知识、柚木好物、推荐商家、信任机制、企业微信承接区和 Footer。
- `styles.css`：完整视觉样式，包含色彩系统、木纹质感、卡片布局、二维码占位、桌面端与移动端响应式。
- `script.js`：绑定所有 `.js-scroll-wechat` 按钮，统一平滑滚动到企业微信区。
- `custom/static-server.mjs`：提供本地静态预览服务，避免直接打开 `file://` 时被浏览器安全策略拦截。
- `custom/notes/v0.1-homepage-demo.md`：记录本轮首轮基线、方案取舍和后续建议。

## 7. 本轮优点
当前方案相比空目录状态，建立了可直接预览的首页基线。

主要优点：

- 可读性：页面结构按业务模块清晰拆分。
- 可维护性：HTML、CSS、JS 分离，后续可迁移到组件化框架。
- 稳定性：不依赖远程资源，不需要安装依赖。
- 转化清晰：企业微信二维码区被放大处理，所有按钮统一导向该区域。
- 风格克制：主色围绕深咖、柚木棕、米白、浅杏、橄榄绿，避免强电商感。

## 8. 本轮缺点与代价
当前方案的主要代价是视觉图片为 CSS 模拟，并不是真实柚木家具摄影图。它适合确认首页方向，但最终商业官网仍需要替换为真实庭院、家具、茶席、工艺、商家案例图片。

另一个缺点是当前为空项目静态基线，并非 React/Next 组件化结构。如果后续仓库恢复为前端工程，需要把这些模块拆成组件并接入现有路由和样式体系。

二维码目前也是 CSS 占位，只用于预留版位，不能实际扫码。

## 10. 本轮结论
本轮确认：在当前空项目状态下，静态单页是最稳妥的首页 demo 基线。页面内容重点应围绕“看懂柚木、连接好物、建立信任、加入社群”，而不是围绕商品交易。

以后应默认继承的经验：

- 首页 CTA 优先导向企业微信承接。
- 产品推荐只讲适合场景与判断依据，不直接卖货。
- 商家展示要用故事、案例、工艺和服务承诺，而不是店铺入口。
