# 版本记录：v1.6-mobile-nav-fix

## 1. 本轮目标

本轮目标是单独修复 V1.4 移动端导航在真实手机查看时出现的遮挡、点不到、抽屉菜单可用性和锚点滚动问题。

本轮只处理移动端 Header 层级、菜单按钮点击热区、抽屉菜单层级、body 锁滚动和锚点滚动偏移，不重做首页视觉，不改 Hero 大结构，不改好物方案内容，不改推荐厂商内容，不改企业微信文案，也不改 GitHub Pages workflow。

## 2. 背景与上下文

本轮承接：
- `custom/notes/v1.4-visual-mobile-experience.md`
- `custom/notes/v1.5-pages-workflow-cleanup.md`

V1.4 新增移动端抽屉菜单，但用户在公网手机端实际查看时反馈：整体页面可以，但移动端导航栏存在被遮盖、点不到的问题。

V1.5 已收口部署链路，本轮不再触碰 `.github/workflows/pages.yml`。

## 3. 问题分析

排查 `index.html`、`styles.css`、`script.js` 后，判断主要风险来自以下几点：

1. 移动端菜单 DOM 位于 `.site-header` 内部，而 `.site-header` 在旧样式里使用 `left: 50%` 加 `transform: translateX(-50%)` 居中。带 `transform` 的父级可能影响内部 `position: fixed` 元素的定位和点击层级。
2. V1.4 中 Header 的基础 `z-index` 仍偏低，`.mobile-menu-shell` 也只有 `z-index: 80`，在真实移动浏览器中可能被 Hero、图片、卡片或其他层叠上下文干扰。
3. 菜单点击后原逻辑是先滚动再关闭菜单，body 处于 `menu-open` 锁滚动状态时，可能导致滚动位置不稳定。
4. 移动端锚点预留距离偏保守，部分栏目标题存在被固定 Header 盖住的风险。

## 4. 候选方案比较

### 方案一：把移动菜单 DOM 移出 Header

优点是可以彻底避开 Header 层叠上下文。缺点是需要修改 HTML 结构，改动比本轮需要更大。

本轮放弃该方案。

### 方案二：保持结构，仅修复移动端层级、定位和滚动顺序

优点是改动集中，不影响桌面端，不重做首页结构。缺点是仍然保留菜单在 Header 内部的结构，未来如果继续复杂化导航，可能还要再整理。

本轮采用该方案。

## 5. 最终决策

最终采用“V1.6 CSS 专项覆盖 + JS 菜单滚动顺序修正”的方案。

关键决策：
- 移动端 Header 取消 `transform` 居中，改为 `left: 12px; right: 12px; width: auto; transform: none;`。
- 移动端 Header 提升到 `z-index: 1000`。
- 菜单按钮提升到 `z-index: 1003`，点击热区不小于 46px。
- 菜单遮罩使用 `z-index: 1100`。
- 菜单抽屉使用 `z-index: 1200`，关闭按钮使用 `z-index: 1300`。
- 菜单面板使用 `height: 100dvh`，允许纵向滚动。
- 点击菜单项后先关闭菜单，再在下一帧滚动到目标锚点。

## 6. 具体实现

### `styles.css`
- 新增 V1.6 移动端导航专项覆盖。
- `body` 增加 `overflow-x: hidden`，降低横向溢出风险。
- 在 `max-width: 900px` 下重设 Header 定位、层级、背景、阴影和点击区域。
- 重设 `.mobile-menu-shell`、`.mobile-menu-backdrop`、`.mobile-menu-panel` 为视口级 fixed 布局。
- 菜单面板加入 `100dvh`、`overflow-y: auto`、`overscroll-behavior: contain` 和安全区 padding。
- 提高 `.mobile-menu-close`、菜单项和 CTA 的点击热区。
- 将移动端锚点预留调整到 124px，680px 以下调整到 132px。

### `script.js`
- 新增 `closeMenuAndScroll(selector)`。
- 点击移动端菜单项、CTA 或页面内锚点时，先关闭菜单并移除 body 锁滚动，再用 `requestAnimationFrame` 平滑滚动到目标。
- 保留遮罩关闭、关闭按钮关闭和 Escape 关闭。
- 好物方案分类标签逻辑未改变。

### 文档
- 更新 `custom/client-preview-note.md`，补充 V1.6 移动端导航修复说明。
- 更新 `custom/launch-checklist.md`，补充 V1.6 移动端导航专项复核项。
- 新增本记录文件。

## 7. Header 层级修复说明

移动端 Header 现在使用 `z-index: 1000`，并取消旧的 `transform`，减少内部 fixed 菜单被父级 transform 影响的风险。

Header 背景改为更不透明的暖米色，并保留轻微边框和阴影，确保在 Hero 图片和浅色背景上都能看清。

## 8. 抽屉菜单修复说明

移动端抽屉菜单现在由三层构成：
- `.mobile-menu-shell`：`z-index: 1100`
- `.mobile-menu-backdrop`：`z-index: 1100`
- `.mobile-menu-panel`：`z-index: 1200`
- `.mobile-menu-close`：`z-index: 1300`

菜单面板固定到视口右侧，使用 `height: 100dvh`，内容可纵向滚动，不依赖 Hero 或 Header 的高度。

## 9. 点击热区修复说明

菜单按钮和关闭按钮均设置为 46px × 46px。移动端菜单项与 CTA 也设置了不低于 46px 的最小高度，方便手指点击。

Logo 宽度在移动端受限，避免压住菜单按钮。

## 10. body 锁滚动处理说明

菜单打开时仍使用 `body.menu-open` 锁定背景滚动。本轮保留该机制，并调整 JS 顺序：点击菜单项后先关闭菜单、解除 body 锁滚动，再执行滚动。

## 11. 锚点遮挡修复说明

移动端 `scroll-padding-top` 和 `scroll-margin-top` 增加到 124px；680px 以下增加到 132px。

JS 中 `getHeaderOffset()` 会根据当前 Header 实际高度计算偏移，并额外预留 26px 呼吸距离。
