# 版本记录：V1.8.2 二级栏目内容索引化重构版

## 1. 本轮目标

本轮要解决的问题，是把二级栏目页从“栏目说明页”调整为“内容索引页”。

优先级高的原因有两个：

- 用户点击“柚木知识”“好物方案”“推荐厂商”后，第一眼应该看到可点击内容，而不是先看到大段栏目介绍。
- V1.8.1 已经把“推荐厂商”内核改成会员站展示体系，如果二级页继续停留在说明型结构，用户路径就会断在中间。

本轮目标属于：

- 结构优化
- 可维护性提升
- 用户路径修正

## 2. 背景与上下文

本轮承接 `custom/notes/v1.8.1-good-category-member-site-and-recommendation-rules.md`。

上一轮已经完成的核心方向是：

- 保留一级导航不变
- 将“推荐厂商”内核改成按好物分类展示品牌或企业会员站
- 保留会员展示不等于平台担保的边界

但在实际复查中发现，多个二级页仍然是“这个栏目是什么”的写法，而不是“这页能点什么”的写法。尤其是：

- `knowledge/index.html`
- `solutions/index.html`
- 五个方案详情页
- `vendors/index.html`

这些页面虽然已经有部分相关链接，但大量正文说明把真正的内容入口压到了中后部，和首页已经建立起来的导航预期不一致。

## 3. 问题分析

表面现象：

- 用户点进二级页后，首先看到的是栏目功能说明。
- 相关入口存在，但层级靠后，无法形成“进入即索引”的感受。

核心矛盾：

- 首页已经在讲“知识、方案、会员站”三类内容路径。
- 二级页却还在讲“栏目存在的意义”，没有直接承担入口职责。

主要原因：

- 早期页面为了补足内容深度，优先使用了长正文结构。
- 后续虽然补了一些 related links，但没有从页面层级上把入口提前。
- `vendors/index.html` 已经完成体系转向，但首屏仍然说明过多，未把五类入口放在第一屏。

次要原因：

- 方案详情页原本更像单篇说明文，入口逻辑不够集中。
- 首页文案和 `data/site-content.js` 口径也需要跟着同步，避免源文本与页面表达脱节。

## 4. 候选方案比较

### 方案 A：保留原正文，只在顶部补一组入口链接

核心思路：

- 不动原有说明段落。
- 在首屏上方加一组跳转入口。

优点：

- 改动最小。
- 内容深度几乎不受影响。

缺点：

- 页面主体仍然是说明页，不是真正的索引页。
- 入口和正文会形成“双主线”，用户阅读重点依然混乱。

放弃原因：

- 不能真正解决“二级页还是在解释自己”的问题。

### 方案 B：整体重排为索引优先，说明后置

核心思路：

- 保留原页面用途和路径。
- 但把首屏和前半段改成“内容入口 / 资料入口 / 样板入口 / 会员站入口 / 咨询入口”。
- 把说明文字压缩并后置，只保留必要边界。

优点：

- 最贴合本轮目标。
- 保持一级导航、页面数量和静态结构不变。
- 可以继续复用现有 `related-links`、`content-body`、`content-cta` 结构，改动集中且可控。

缺点：

- 多个页面需要重写正文顺序。
- 需要额外同步首页和文档口径。

最终选择：

- 采用方案 B。

## 5. 最终决策

本轮最终采用“索引优先，说明后置”的实现方式。

关键设计决策：

- 不新增一级栏目，不恢复文章、案例、咨询表单为一级导航。
- 不新增后台、登录、商城、支付、数据库。
- 不改 `forms/form.js`、GitHub Actions workflow、sitemap。
- 直接复用现有页面路径和 CSS 结构，避免扩散改动。
- 在方案详情页保留必要正文，避免变成只有按钮的空壳页。
- 在会员站相关页面继续保留“不代表平台担保”的边界表达。

## 6. 具体实现

本轮修改文件如下：

- `knowledge/index.html`
- `solutions/index.html`
- `solutions/whole-decoration.html`
- `solutions/flooring.html`
- `solutions/outdoor.html`
- `solutions/tea-room.html`
- `solutions/furniture.html`
- `vendors/index.html`
- `index.html`
- `data/site-content.js`
- `custom/client-preview-note.md`
- `custom/launch-checklist.md`

各文件作用：

- `knowledge/index.html`
  改为柚木知识索引页，首屏直接展示六类知识入口和延伸阅读入口。
- `solutions/index.html`
  改为柚木好物资料索引页，首屏直接展示五类好物入口，并补充每类对应的资料、样板案例结构、会员站样板或咨询入口。
- 五个方案详情页
  全部改为资料入口页，不再作为长篇栏目说明文使用。
- `vendors/index.html`
  进一步改为“先看五类入口，再看规则说明”的会员站索引页。
- `index.html`
  只轻量同步首页“推荐厂商”模块文案，不改首页结构。
- `data/site-content.js`
  同步首页相关内容源口径，避免“好物方案”和“会员站”表达仍停留在旧说法。
- 两份说明文档
  补齐 V1.8.2 版本口径。

## 7. 本轮优点

相比之前，本轮方案的主要收益是：

- 二级页终于承担起“入口页”的职责。
- 用户点进页面后，第一眼就能看到可继续点击的内容。
- 首页建立的导航预期和二级页实际内容结构更加一致。
- 继续保持了一级导航稳定，没有因为索引化重构而新增栏目。
- 继续沿用了现有静态结构，没有把项目带向后台化或交易化。
- 风险边界表达仍然清楚，避免把会员站展示写成平台背书。

## 8. 本轮缺点与代价

本轮仍有明显代价和限制：

- 庭院户外、茶室会客方向仍然没有真实会员站样板，只能先保留资料入口和待补充说明。
- 方案详情页虽然已改成入口页，但还不是最终长期版式，后续仍可能继续精修文字与层级。
- 首页 `script.js` 目前并未驱动所有二级页内容，因此内容源同步只覆盖了首页相关字段。
- 当前所有入口仍然是静态资料入口，不解决真实资料缺口本身。

暂时接受的妥协：

- 不为本轮新增样式系统。
- 不为索引页引入新的交互逻辑。
- 继续使用样板案例结构和样板会员站逻辑，而不伪造真实内容。

## 10. 本轮结论

本轮最重要的结论是：

- 二级页索引化不需要新增任何栏目，也不需要改动系统架构，只要重排页面层级，就能显著改善用户进入后的第一屏体验。

被证实的判断：

- 用户真正需要的是“点进来就能继续点”的路径感，而不是继续读栏目解释。
- 现有静态结构和页面组件足以支持这次重构，不需要为了索引化引入后台或复杂交互。

本轮形成的新约束：

- 以后继续补二级页内容时，应优先保持“入口在前，说明在后”的结构。
- 会员站相关页面必须继续保留“不代表平台担保”的清晰边界。

## 11. 下一轮建议

1. 等 GitHub Pages 部署完成后，逐页抽查 `knowledge/index.html`、`solutions/index.html`、五个方案详情页和 `vendors/index.html` 的公网首屏表现。
2. 结合真实资料补齐进度，优先决定庭院户外和茶室会客方向是否具备补会员站样板的条件。

补充说明：

- 本轮没有恢复文章、案例、咨询表单为一级导航。
- 本轮没有改 `forms/form.js`、workflow 和 sitemap。
