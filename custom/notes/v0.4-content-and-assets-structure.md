# 版本记录：v0.4-content-and-assets-structure

## 1. 本轮修改目标
本轮目标是在 V0.3 高级视觉首页基础上，整理“内容数据结构、示例内容、图片素材清单和真实素材替换说明”，让首页从静态写死内容进一步升级为更易维护的内容型官网 demo。

本轮不新增后台、登录、商城、支付或数据库，不改变现有静态页面主结构。

## 2. 实际修改文件
- `data/site-content.js`：新增首页统一内容数据源。
- `assets/README.md`：新增真实图片与二维码素材需求清单。
- `index.html`：接入 `data/site-content.js`，保留当前静态页面结构。
- `script.js`：读取 `window.YUXI_SITE_CONTENT`，并标记当前页面已接入内容源。
- `custom/notes/v0.4-content-and-assets-structure.md`：新增本轮记录。

## 3. 新增内容数据文件说明
`data/site-content.js` 当前包含：

- SEO 信息
- 品牌信息
- 导航菜单
- Hero 文案与行动按钮
- 三大价值
- 柚木知识内容
- 柚木好物内容
- 推荐厂商内容
- 推荐厂商筛选标准

当前项目仍是静态 HTML，因此 V0.4 没有把整个页面改为 JS 动态渲染，以避免破坏已有可预览结构。后续如果接入 React/Next 或拆组件，可以直接以该文件作为内容源。

## 5. 哪些内容是示例，哪些上线前必须替换
示例内容：

- 三个推荐厂商名称与介绍：`山木工坊`、`木语空间`、`柚木世家` 均为展示 demo 示例，不代表真实资质或合作关系。
- 柚木知识、好物推荐和筛选标准为首页展示文案示例。
- Footer 电话、邮箱、地址仍为占位信息。
- 公众号二维码仍为占位图。
- 多数视觉图片仍为 SVG 占位图。

上线前必须替换：

- 企业微信二维码需确认是否为最终投放二维码。
- 推荐厂商名称、案例、产品、交付和用户反馈需要替换为真实内容。
- Footer 联系方式和地址需要替换为真实信息。
- Hero、知识、好物、厂商图片建议替换为真实授权图片。
- 公众号二维码需要替换为真实二维码。

## 6. 真实图片替换建议
真实图片建议优先放在 `assets/` 下，并沿用 `assets/README.md` 中的建议文件名。

替换方式：

- Hero：替换 `index.html` 中 `.hero-image` 的 `src`。
- 知识区：替换各 `.knowledge-card img` 的 `src`。
- 好物区：替换各 `.goods-card img` 的 `src`。
- 推荐厂商区：替换各 `.merchant-card > img` 的 `src`。
- Footer 公众号二维码：替换 `.mini-qr` 的 `src`。

## 7. 企业微信二维码替换位置
企业微信二维码位于 `index.html` 的 `.qr-panel` 内：

```html
<img class="qr-code-image" src="./assets/wecom-qr.jpg" alt="扫码添加柚喜顾问企业微信二维码" />
```

如果后续改用 PNG，可将真实图片放到 `assets/wecom-qr.png`，再调整该 `src`。
