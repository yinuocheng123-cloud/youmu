# 版本记录：v1.20.4-good-things-category-tabs-page-length

## 1. 本轮目标

本轮目标是把 `solutions/index.html` 从“六类全部长篇铺开”调整为“同一入口内的六类栏目切换”，让用户进入柚木好物后先选择分类，再只阅读当前分类内容。

本轮属于页面长度优化与轻量交互增强，不新增第二入口，不新增独立页面，不修改 Header 导航结构，不修改 sitemap、workflow、`forms/form.js`，不修改 30 个好物文章 URL 和正文，也不修改知识文章正文。

## 2. 背景与上下文

本轮承接 `v1.20.3-good-things-hero-slim-category-nav` 和 `v1.20.1-good-cultural-anchor-rollback-fix`。

V1.20.3 已经删除首屏解释区，并把六类二级导航前置。但页面下方仍然一次性铺开六类内容，用户需要不断向下滚动才能看完。V1.20.1 已经确认柚木文创锚点统一为 `good-cultural`，本轮继续保留该锚点，不执行旧恢复命令。

## 3. 问题分析

问题表面是页面太长，核心是六类内容虽然属于同一个入口，但展示方式仍像六个大区连续堆叠。读者点击某个分类后，最好只看到该分类下的说明、代表文章、更多内容和相关参考。

因此本轮不改变内容关系，只改变展示方式：六类仍在同一页面，锚点仍存在，但默认只展开当前栏目。

## 4. 候选方案比较

方案一：继续保留六类长铺开，只压缩卡片样式。

优点是无需 JavaScript；缺点是页面长度仍然偏长，手机端体验改善有限。

方案二：拆成六个独立页面。

优点是每页更短；缺点是会破坏“柚木好物单入口”的方向，也会新增 URL 和维护成本。

方案三：保留同页六类锚点，用轻量 JavaScript 做栏目切换。

优点是保留单入口和原有锚点，同时显著降低页面长度；缺点是需要维护少量交互逻辑。最终选择方案三。

## 5. 最终决策

页面仍只有一个入口：柚木好物。

六类内容改为同页栏目切换：

- 默认打开 `#good-furniture`
- 点击分类导航或分类卡片时，切换到对应栏目
- URL hash 同步变化
- 直接访问 `#good-cultural` 等锚点时，自动显示对应栏目
- 无效 hash 回落到柚木家具

## 6. 具体实现

本轮修改：

- `solutions/index.html`：为六类导航和分类卡片增加 `data-good-category-tab`；为六个分类 section 增加 `good-category-panel`，默认家具增加 `is-active`，其他栏目默认 `aria-hidden="true"`。
- `styles.css`：新增 `good-category-panel` 隐藏规则和 `is-active` 展示规则；为当前分类导航和分类卡片增加选中态。
- `script.js`：新增栏目切换逻辑，支持点击切换、hash 初始化、hashchange 和浏览器前进/后退；同时修正动态好物菜单中的旧 `#good-creative` 为 `#good-cultural`。
- `custom/check-content-depth.mjs`：新增栏目切换类名检查，并拦截 `script.js` 中的 `good-creative`。
- `custom/check-release-readiness.mjs`：同步发布预检规则。

## 7. 本轮优点

页面长度明显降低，默认只显示一个分类内容。六类仍然属于同一个“柚木好物”入口，用户可以通过栏目导航快速切换，不需要在六个长区块之间不断下拉。

`good-cultural` 被继续保留为柚木文创唯一公开锚点，检查脚本也开始拦截 `script.js` 中的旧锚点别名。

## 8. 本轮缺点与代价

本轮引入了少量 JavaScript 状态逻辑。若浏览器禁用 JavaScript，页面会保留默认家具栏目，不会展开全部六类。这个代价可以接受，因为站点本身已经使用脚本处理导航和锚点交互。

本轮没有重写 30 篇好物文章正文，也没有进一步压缩每个栏目内部的 5 篇代表文章。

## 10. 本轮结论

柚木好物仍应坚持单入口。六类不需要全部长篇铺开，更适合做成同页栏目切换：入口统一，内容可扩展，页面长度更可控。

## 11. 下一轮建议

1. 部署后用手机端重点检查分类切换的点击热区和 hash 直达。
2. 如果仍觉得栏目内部偏长，可继续压缩“更多内容”和“相关参考”的展示密度。
3. 后续新增真实素材时，仍按六类归入当前栏目，不新增第二套入口。
