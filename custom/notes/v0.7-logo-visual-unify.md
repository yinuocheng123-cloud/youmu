# 版本记录：v0.7-logo-visual-unify

## 1. 本轮目标
本轮目标是在 V0.6 首页结构与交付检查已稳定的基础上，将网站可见品牌标识统一升级为“喜字方向 Logo”，并补齐横版 Logo、图标版 Logo、深色版、单色版和 favicon。

本轮属于品牌视觉统一，不新增后台、登录、商城、支付、数据库，不处理真实厂商资料真实性，不替换真实厂商案例，不初始化 Git，不 commit、push、deploy。

## 2. 背景与上下文
本轮承接 `custom/notes/v0.6-page-sync-and-delivery-check.md` 的静态首页收口结论：保持首页结构、导航、Hero 文案、核心区块和企业微信承接路径稳定，只做明确必要的小幅视觉同步。

同时补读了 `custom/notes/v0.7-logo-mark-update.md`。上一轮已把用户提供的位图 Logo 接入 header 和 Footer，但仍缺少网站可复用 SVG 资源、favicon、资源说明和甲方预览说明。本轮延续“喜字 Logo”方向，并把位图预览升级为站点可用 SVG 资产组。

## 3. 问题分析
旧实现主要问题是品牌资源不完整：页面使用的是单个位图文件，无法覆盖 favicon、深色背景、单色应用和后续二维码中间标识等场景。

核心矛盾不是重做首页，而是在不破坏 V0.6 已确认首页结构的前提下，让品牌识别在页眉、页脚、小图标和说明文档中保持一致。

## 4. 候选方案比较
方案一：继续使用现有 `yuxi-logo-mark.png`。
优点是改动最少；缺点是缺少横版组合、favicon、深色版和单色版，后续扩展成本高。

方案二：新增一组 SVG Logo，并替换页面可见品牌位。
优点是清晰、轻量、可缩放，适合网站头部、Footer 和 favicon；缺点是当前仍是 demo 方向稿，不等同于专业设计师最终商标稿。

本轮选择方案二。

## 5. 最终决策
新增一组以单个“喜”字为核心的 SVG 品牌资源。图形保留喜字识别度，上下两个“口”的下边缘使用轻微微笑弧线，边角和收笔做圆润处理，色彩控制在柚木金、木棕、深咖、暖米和柔金体系内。

页面上使用 `logo-yuxi-horizontal.svg` 统一 header 与 Footer；二维码承接区增加小号 `logo-yuxi-mark.svg`；浏览器 favicon 使用 `favicon.svg`。

## 6. 具体实现
- `assets/logo-yuxi-horizontal.svg`：新增横版组合 Logo，用于网站头部和 Footer。
- `assets/logo-yuxi-mark.svg`：新增浅色背景喜字图标，用于小图标和二维码附近品牌标识。
- `assets/logo-yuxi-mark-dark.svg`：新增深色背景暖金版本。
- `assets/logo-yuxi-mark-mono.svg`：新增单色版本。
- `assets/favicon.svg`：新增浏览器标签页图标。
- `index.html`：配置 favicon；顶部导航、Footer 改用横版 Logo；企业微信二维码区新增小号喜字图标。
- `styles.css`：调整横版 Logo 的尺寸、浅色承托、深色页眉和 Footer 可读性、移动端显示比例。
- `data/site-content.js`：同步品牌 Logo 路径与 demo 版说明。
- `assets/README.md`：新增“品牌 Logo 资源说明”。
- `custom/client-preview-note.md`：新增“Logo 说明”。

## 7. 本轮优点
品牌资源从单个位图升级为可复用 SVG 资产组，页眉、页脚、favicon 和二维码承接区附近的品牌视觉保持一致。

Logo 仍保持克制和现代东方气质，没有加入叶子、家具、房子、门窗等具象图形，也没有走婚庆双喜方向。

## 8. 本轮缺点与代价
当前 Logo 仍为网站 demo 版方向稿，非最终商标注册稿，也未经过专业字体路径化、商标近似检索、VI 延展和印刷测试。

横版 SVG 内部使用浏览器安全字体策略显示中文，未做文字路径化；如果后续要用于正式商标或离线印刷，应由专业设计师输出路径化矢量稿。

## 11. 下一轮建议
1. 甲方确认 Logo 方向后，交由专业设计师进行最终矢量精修、文字路径化和品牌规范整理。
2. 若后续要做公众号头像、企业微信头像或二维码中间标识，可优先沿用 `logo-yuxi-mark.svg`，再按实际平台尺寸导出 PNG。
3. 正式上线前复核 favicon、Logo 文件缓存、二维码是否为最终投放版本。
