# 版本记录：v1.0-knowledge-products-and-nav

## 1. 本轮目标

本轮目标是在 V0.9 已完成真实内容深度填充与合规素材替换的基础上，继续优化首页信息架构、顶部导航命名、知识分类入口和未来商业承接方向。

本轮属于信息架构优化、文案结构优化和轻量交互优化，不新增后台、登录、商城、支付、数据库，不初始化 Git，不 commit、push、deploy。

## 2. 背景与上下文

本轮承接 `custom/notes/v0.9-real-content-and-compliant-assets.md`。V0.9 已经将首页图片从 SVG 占位替换为开放授权图库素材，并建立 `custom/image-source-log.md`，同时明确图片只是 demo 预览素材，不代表真实厂商案例或最终上线素材。

修改前首页结构已经稳定：Hero、三大价值、柚木知识、柚木好物、推荐厂商、推荐标准、企业微信社群、Footer。本轮不重做首页结构，只在既有区块内调整导航命名、知识分类、好物方案和企业微信承接口径。

任务开始时按项目规则尝试执行 `git pull --ff-only`，但当前目录不是 Git 仓库，命令返回 `fatal: not a git repository`。因此本轮没有发生 Git 初始化、提交、推送或部署。

## 3. 问题分析

V0.9 的主要问题是：导航仍有“项目介绍”“柚木好物”“加入我们”等偏项目方视角或早期 demo 视角的名称，用户进入页面时不容易马上理解每个栏目解决什么问题。

另外，知识区虽然已有内容，但仍像几张文章卡片，不够像长期运营的知识入口；好物区也仍偏单品好物，尚未承接甲方未来重点方向“柚木整装”和“柚木地板”。

核心矛盾是：需要让页面更利于后续商业承接，但不能变成商城，不能写价格、购物车、立即购买，也不能承诺真实交付能力或编造真实厂商案例。

## 4. 候选方案比较

### 方案一：只改导航名称，不改栏目内部

优点：改动最小，风险最低。

缺点：用户点进栏目后仍看不到知识分类和整装、地板方向，无法完成本轮目标。

结论：放弃。它只能解决表层命名，不能解决信息架构。

### 方案二：新增独立知识页和方案页

优点：更接近未来完整网站结构。

缺点：本轮明确要求不新增复杂功能、不重做首页结构；独立页面会增加维护范围，也会分散企业微信承接路径。

结论：放弃。本轮只做首页锚点和轻量下拉。

### 方案三：在首页原区块中做分类化升级

优点：不改变首页模块顺序；用户能在首页快速看到知识分类、方案分类和承接路径；移动端压力可控；所有 CTA 仍进入企业微信。

缺点：当前仍是静态首页，分类只是入口和方向表达，不是完整知识库或方案详情页。

结论：采用。该方案最符合当前阶段“继续给甲方预览”的要求。

## 5. 最终决策

最终采用“导航命名优化＋柚木知识轻量下拉＋首页知识分类卡＋好物方案五类入口”的方案。

关键决策：

- 顶部导航统一为“认识柚喜｜柚木知识｜好物方案｜推荐厂商｜社群交流”。
- “柚木知识”增加桌面悬停/聚焦下拉和移动端点击展开。
- 知识区改为 6 类：柚木入门、选购避坑、材质工艺、空间应用、保养维护、常见问题。
- 好物区升级为“柚木好物与空间方案”，加入柚木整装、柚木地板、庭院户外、茶室会客、家具好物。
- 柚木整装和柚木地板作为未来重点承接方向，但只写“方向”“咨询”“应用参考”，不写交付承诺。
- 推荐厂商区继续保留样板位表达，不出现真实厂商名称。

## 7. 本轮优点

- 导航从项目方视角转为用户视角，入口更清楚。
- 知识区从文章卡片升级为长期知识分类入口，更适合后续运营和 GEO 内容沉淀。
- 好物区从单品展示升级为空间方案入口，承接柚木整装和柚木地板两个未来重点方向。
- 企业微信承接更明确，增加知识、整装地板、厂商共创三个核心沟通入口。
- 推荐厂商区继续保留样板位边界，没有虚构真实厂商或真实案例。
- 没有新增图片，继续复用 V0.9 已记录来源的合规素材。

## 8. 本轮缺点与代价

当前知识分类和好物方案仍是首页入口级表达，不是完整知识库、方案详情页或真实销售链路。用户如果需要更细信息，仍然需要通过企业微信承接。

柚木整装和柚木地板虽然已经进入页面表达，但当前没有真实产品参数、交付能力说明、安装售后说明或授权案例图片。因此本轮只能用“方向”“咨询”“应用参考”表达，不能写成已具备全国交付或全案落地能力。

移动端导航因为加入了二级分类，比 V0.9 更复杂。为保持稳定，本轮只给“柚木知识”做下拉，没有给“好物方案”做顶部下拉，而是在栏目内部呈现五类方案。

## 10. 本轮结论

V1.0 已经把首页从“内容更丰满的预览版”推进到“信息架构更清楚、分类更适合运营、好物承接方向更明确”的版本。导航、知识分类、好物方案和企业微信承接路径已经形成更清晰的用户路径。

本轮继承 V0.9 的合规素材策略和样板位边界，没有新增图片，也没有把整装、地板方向包装成真实交付能力。

## 11. 下一轮建议

1. 甲方确认是否接受“好物方案”作为导航名称，以及是否需要在后续阶段为“柚木整装”和“柚木地板”建立独立详情页。
2. 若进入真实销售承接阶段，优先补充柚木地板参数、安装范围、售后边界和整装案例授权。
