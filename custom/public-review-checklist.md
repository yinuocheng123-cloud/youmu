# 人工验收清单

文件说明：本文件用于正式发布前的人工作业验收。  
功能说明：把站点最关键的人工检查项整理成可勾选清单，便于上线前逐项确认。

结构概览：
  第一部分：首页验收
  第二部分：知识页验收
  第三部分：方案页验收
  第四部分：厂商会员站验收
  第五部分：案例页验收
  第六部分：表单页验收
  第七部分：技术验收

## 1. 首页验收

- [ ] 首页能正常打开
- [ ] 主导航只有五项入口
- [ ] 手机端菜单能正常打开和关闭
- [ ] 首页 CTA 没有误导到错误入口
- [ ] 首页没有交易承诺
- [ ] 首页 Hero 图、Logo、二维码图显示正常
- [ ] Footer 联系方式与二维码展示符合当前公开策略

## 2. 知识页验收

- [ ] 中文显示正常
- [ ] 页面不是骨架页
- [ ] 页面有来源说明或来源绑定提醒
- [ ] 页面没有夸大表达
- [ ] 页面没有把知识说明写成认证结论

## 3. 方案页验收

- [ ] 柚木整装表达清楚
- [ ] 柚木地板表达清楚
- [ ] 页面不承诺真实交付
- [ ] 页面不写平台担保
- [ ] 咨询入口合理
- [ ] 图片、说明、边界提示一致

## 4. 厂商会员站验收

- [ ] 页面表达为“会员站”，不是推荐名单
- [ ] 五项确认提示完整
- [ ] 会员展示不等于平台担保
- [ ] 样板页不会被误解为真实入驻
- [ ] 总入口页、样板页、申请页的边界口径一致

## 5. 案例页验收

- [ ] 明确属于样板案例
- [ ] 不写成功案例
- [ ] 不写真实成交案例
- [ ] 保留授权说明或授权提醒

## 6. 表单页验收

- [ ] 咨询摘要能正常生成
- [ ] 厂商申请摘要能正常生成
- [ ] 静态表单说明清楚
- [ ] 没有提交成功或后台承诺
- [ ] 承接方式提示与真实流程一致

## 7. 技术验收

- [ ] 全站链接检查通过
- [ ] 编码检查通过
- [ ] 内容深度检查通过
- [ ] 正式发布预检脚本已运行
- [ ] GitHub Pages 二级页面可打开
- [ ] sitemap 不包含 `custom/`、截图或记录文件

人工验收备注：

- 如果预检脚本报出占位联系方式、二维码待替换或高风险表达，先记录为上线阻塞项，不要为了“通过检查”而伪造真实资料。
