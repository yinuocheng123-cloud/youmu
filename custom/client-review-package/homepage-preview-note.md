# 首页预览版说明

## 1. 首页结构说明
当前首页包含以下模块：

- Hero 首屏。
- 三大价值。
- 柚木知识。
- 柚木好物与空间方案。
- 推荐厂商。
- 推荐标准。
- 企业微信社群。
- Footer。

这些模块共同构成从“认识品牌”到“看懂柚木”再到“进入企业微信承接”的首页路径。

## 2. 首页定位说明
当前首页不是普通家具商城，也不是直接卖货页面。

首页的核心路径是：

看懂柚木 → 连接好物方案 → 了解推荐厂商 → 进入企业微信社群。

这样做的目的，是先建立用户对柚木知识、好物方案和推荐厂商机制的信任，再通过企业微信进行更细的沟通、咨询和合作承接。

V1.0 已将首页从单品好物扩展到柚木整装、柚木地板和空间方案，但仍保持低电商感，不做直接商城交易，不展示价格、购物车或“立即购买”。

V1.1 已补齐首页内容，并优化点击路径。用户可以从认识品牌、学习知识、查看方案、了解推荐厂商，最终进入企业微信社群交流。当前首页所有主要按钮和导航均有明确结果，不进入不存在页面。

## 3. 企业微信承接说明
当前首页的主要按钮都会引导到企业微信二维码区。

用户可以先添加柚喜顾问，再根据需求进入：
- 爱好者社群。
- 柚木选购咨询。
- 厂商合作沟通。
- 好物或案例进一步了解。
- 柚木整装方向或柚木地板选购咨询。

这套路径更适合当前阶段的内容平台与社群承接定位，避免一开始就做成价格驱动的家具商城。

## 4. 当前页面仍为预览版
当前页面仍是预览版，以下内容后续需要替换或确认：

- V0.9 已使用开放授权图库素材替换原 SVG 占位图，仍建议后续替换为甲方真实授权图片。
- 真实厂商。
- 真实案例。
- 真实联系方式。
- 企业微信二维码最终版。

- 上线所需备案信息。

资料真实性、授权和最终投放状态将在后续真实素材替换阶段统一处理。

## 5. V1.2 公网预览适配说明
当前首页已按 GitHub Pages 项目页地址进行适配，建议预览地址为：

`https://yinuocheng123-cloud.github.io/youmu/`

本轮只处理公网预览所需的静态站点文件、相对路径、基础 SEO 和上线前检查说明，不新增后台、登录、商城、支付或数据库。

当前公网预览用于确认首页整体效果，不代表正式上线版本。正式上线前仍需统一确认真实图片、真实厂商、真实案例、二维码最终版、联系方式、备案信息和授权状态。

GitHub Pages 手动开启步骤见 `custom/github-pages-guide.md`；发布目录请在 GitHub 界面选择 `/ (root)`，不要理解为 `/root` 路径。

如果分支发布方式仍然 404，V1.3 已新增 GitHub Actions Pages 部署 workflow。推送后请到 GitHub 仓库 `Actions` 页面查看 `Deploy static site to GitHub Pages` 是否成功。

## 6. V1.4 首页视觉与移动端说明
V1.4 保持原有首页模块顺序不变，主要调整用户第一眼看到和手机端使用时的体验：

- Hero 首屏调整为左侧品牌文案、右侧生活方式图片，减少暗色大图对文字可读性的影响。
- 顶部 Logo 在 Header 中使用更简洁的喜字 mark 加品牌名，Footer 继续保留横版组合 Logo。
- 桌面端保留柚木知识与好物方案的下拉入口；手机端改为菜单按钮和抽屉式导航。
- 好物方案增加分类标签，默认突出柚木整装和柚木地板，其他分类包括庭院户外、茶室会客和家具好物。
- 企业微信承接区调整为更明确的深色转化区域，并补充手机端长按识别或截图识别二维码的提示。

这些调整的目的不是把网站改成商城，而是让用户更快理解：先看懂柚木，再进入方案、厂商和社群承接。页面仍不展示价格、购物车或“立即购买”。
