# 柚喜饰界预览说明

文件说明：本文档用于向项目方说明当前站点的预览定位与发布前边界。  
功能说明：帮助阅读者快速理解当前已经完成什么、还缺什么，以及为什么当前仍不应被视为正式上线站。  

结构概览：
  第一部分：当前版本定位
  第二部分：当前已完成内容
  第三部分：当前仍未完成内容
  第四部分：V1.8.1 新增说明
  第五部分：V1.8.2 新增说明
  第六部分：V1.8.3 新增说明

## 1. 当前版本定位

当前网站仍是“柚喜饰界”静态内容型预览站。  
它用于确认以下内容：

- 首页结构和视觉方向
- 五项一级导航结构
- 柚木知识、好物方案、按好物分类展示的会员站和社群承接的内容路径
- 正式发布前需要补齐的资料与人工验收流程

当前网站不是：

- 商城
- 后台系统
- 登录系统
- 支付系统
- 数据库驱动站点
- 平台担保型交易站

## 2. 当前已完成内容

当前版本已经完成：

- 首页整体结构
- 一级导航与移动端菜单
- 柚木知识页基础内容
- 好物方案页基础内容
- 按好物分类展示会员站的总入口与三个样板页
- 会员站申请摘要页
- 链接、编码、内容深度等基础检查脚本
- “推荐厂商”向“按好物分类展示会员站”的页面内核调整
- 正式发布前资料清单、人工验收清单、来源绑定清单、会员站资料模板和发布预检脚本

## 3. 当前仍未完成内容

当前仍未完成、且不能假装已经完成的内容包括：

- 最终企业微信二维码确认
- 最终公众号二维码确认
- 真实联系电话确认
- 真实联系邮箱确认
- 备案信息确认
- 真实品牌 / 企业会员站资料确认
- 真实案例资料确认
- 图片授权确认
- 逐页来源绑定确认
- 庭院户外会员站样板
- 茶室会客会员站样板

说明：

- 这些内容没有确认前，当前站点只适合作为预览站和上线前整理底稿。
- 没有真实资料时，不应把样板页写成真实入驻、真实案例或平台担保内容。

## 4. V1.8.1 新增说明

V1.8.1 的重点不是扩功能，而是把“推荐厂商 / 厂商会员站”进一步升级为“按柚木好物分类展示品牌或企业会员站”，并补齐内部推荐规则说明。当前已新增或更新：

- 首页厂商模块的五类好物会员站入口
- `vendors/index.html` 的五类入口与温和展示规则说明
- `forms/vendor-apply.html` 的好物分类会员站申请口径
- `custom/vendor-member-site-material-template.md`
- `custom/vendor-member-recommendation-rules.md`

当前结论：

- 网站结构已经基本稳定。
- 但距离正式发布仍差“真实资料补齐”和“人工复核闭环”。
- 当前更适合继续按清单补资料，而不是继续扩页面。

## 5. V1.8.2 新增说明

V1.8.2 的重点不是新增栏目或功能，而是把二级栏目页从“栏目说明页”调整成“内容索引页”。当前已完成：

- `knowledge/index.html` 改为柚木知识索引页，首屏直接展示入门、选购、材质、空间、保养、问答和延伸阅读入口。
- `solutions/index.html` 改为柚木好物资料索引页，首屏直接展示整装、地板、户外、茶室会客和家具好物五类入口。
- 五个方案详情页改为资料入口页，优先展示知识入口、样板案例结构、会员站样板或待补充说明、咨询入口和边界提示。
- `vendors/index.html` 进一步改成“先看分类入口，再看规则说明”的会员站索引页。
- 首页“推荐厂商”模块只做轻量口径同步，继续保持一级导航不变。

本轮仍然没有新增后台、登录、商城、支付、数据库，也没有恢复文章、案例、咨询表单为一级导航。当前站点更适合作为可点开的内容索引与资料预览站，而不是正式交易或平台担保站。

## 6. V1.8.3 新增说明

V1.8.3 的重点是修复移动端全站导航可用性，并将普通用户承接从“填写在线页面”调整为“社群交流 / 添加柚喜顾问”。当前已完成：

- 普通用户前台入口不再主动引导进入 `forms/consult.html`。
- 首页、知识页、方案页、厂商页、文章页和样板案例页的相关 CTA 已统一引导到首页社群交流区。
- `forms/consult.html` 暂时保留为备用说明页，只说明当前推荐添加柚喜顾问，不再作为主要承接页面。
- `forms/vendor-apply.html` 继续保留，仍用于申请建立好物分类会员站资料收集。
- 移动端导航已复核全站脚本路径，二级页面统一加载 `../script.js`，内容页缺少移动端菜单结构时由全站脚本补齐。

本轮仍然没有新增后台、登录、商城、支付、数据库，也没有修改 GitHub Actions workflow、sitemap 或 `forms/form.js` 的表单摘要逻辑。
