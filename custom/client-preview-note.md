# 柚喜饰界官网首页预览说明

文件说明：本文件用于向甲方说明“柚喜饰界｜柚木爱好者乐园”官网首页当前预览版本。
功能说明：概括当前版本定位、已完成内容、仍为示例的部分、甲方下一步需提供资料和后续流程。

结构概览：
  第一部分：当前版本说明
  第二部分：当前完成范围
  第三部分：仍需补充内容
  第四部分：预览重点与后续步骤

## 当前版本说明
这是“柚喜饰界官网首页设计 demo／预览版”，用于确认首页方向、信息结构、视觉气质和企业微信社群承接路径。

当前版本不是最终上线稿，暂不处理真实资料真实性，也不核验真实厂商、真实案例和真实图片。

## Logo 说明
当前版本已加入“喜字方向”的品牌标识，用于统一网站头部、Footer、favicon 和二维码承接区附近的品牌视觉。该 Logo 是网站预览版 Logo／品牌方向稿，方向为“高级、温润、现代东方、带有微笑感的喜字图形”；后续如甲方确认，可再交由专业设计师做最终矢量精修和商标规范。

## 甲方预览确认包
当前已整理独立预览确认包：`custom/client-review-package/`。该目录集中说明 Logo 方向、首页预览版定位、下一步真实素材清单，并保留桌面端和手机端预览截图，便于甲方集中确认方向。

若方向已确认，下一步可进入真实素材替换阶段；真实厂商、真实图片、联系方式、二维码最终版和授权情况仍需在替换前统一确认。

## 当前网站定位
柚喜饰界定位为：

柚木知识平台＋好物方案＋推荐厂商展示＋企业微信社群承接入口。

页面整体不是普通家具商城，不做购物车、首页价格、立即购买、下单、支付或会员登录功能。

## V0.9 内容与素材说明
当前版本已在不改变首页结构的前提下，补充了柚木知识、好物推荐、推荐厂商样板位和企业微信社群承接区的说明文案，并使用开放授权图库素材替换原有 SVG 占位图。

这些素材用于继续预览网站气质和内容密度，不代表最终上线素材，也不构成真实厂商背书。正式上线前，仍建议使用甲方自有图片、真实授权厂商图片和最终确认后的联系方式、二维码与备案信息。

## V1.0 导航与方案说明
当前版本已完成导航命名优化，导航从“项目介绍／加入我们”等项目方视角，调整为“认识柚喜／社群交流”等用户视角，并统一为“认识柚喜｜柚木知识｜好物方案｜推荐厂商｜社群交流”。

同时，首页已增加柚木知识分类入口，并将柚木好物升级为“柚木好物与空间方案”。其中“柚木整装方向”和“柚木地板选购咨询”作为未来重点承接方向加入网站表达，但当前仍保持低电商感，不做购物车、价格、立即购买或直接交易。

## V1.1 交互闭环说明
当前版本已完成首页交互闭环处理，首页所有主要导航、下拉分类、按钮和方案入口均有明确指向，不再出现点击进入不存在页面的情况。知识类入口指向首页对应分类或企业微信承接区，方案和厂商相关按钮统一引导到企业微信二维码区。

本轮同时补齐了“认识柚喜”、柚木知识分类、柚木整装、柚木地板、推荐厂商样板位、推荐标准和社群交流说明，让首页从“能看”进一步变成“能点、能读、能咨询”的预览版本。

## V1.2 GitHub Pages 公网预览说明
当前项目已进入 GitHub Pages 公网预览适配阶段，预览地址暂按 `https://yinuocheng123-cloud.github.io/youmu/` 准备。

公网预览用于甲方确认整体效果，不代表正式上线版本。图片、真实厂商、真实案例、最终二维码、联系方式、备案信息和授权状态，仍需在正式上线前统一确认。

本轮补齐了 `.nojekyll`、`404.html`、`robots.txt`、`sitemap.xml` 和 `custom/github-pages-guide.md`。当前 `robots.txt` 采用“公网预览，不主动收录”的策略。
如果 GitHub 页面里暂时找不到 Pages 入口，请优先查看 `custom/github-pages-guide.md`，按仓库 `Settings -> Code and automation -> Pages` 路径开启，并确认 Folder 选择 `/ (root)`。

## V1.3 GitHub Actions Pages 部署说明
由于分支发布方式设置后公网地址仍返回 404，当前已准备 GitHub Actions Pages 部署方案。新增 `.github/workflows/pages.yml`，用于在推送 `master` 后自动打包静态站点并部署到 GitHub Pages。
如采用该方式，请在仓库 `Settings -> Pages` 中将 Source 设为 `GitHub Actions`，并到 `Actions` 页面查看 `Deploy static site to GitHub Pages` 是否运行成功。

## V1.4 首页视觉与移动端体验说明
当前版本已在不重做首页模块顺序的前提下，优化了首页首屏、顶部导航、好物方案分类和企业微信承接区。首屏从整屏暗色背景调整为左侧清晰文案、右侧生活方式图片的结构，更适合甲方和移动端用户快速理解“柚木知识平台＋好物方案＋推荐厂商＋企业微信社群入口”的定位。

移动端已改为右侧菜单按钮和抽屉式导航，包含柚木知识分类、好物方案分类、推荐厂商和社群入口。好物方案中继续突出柚木整装和柚木地板两个重点方向，但仍保持低电商感，不展示价格、购物车或“立即购买”。企业微信二维码区也补充了手机端长按识别或截图识别提示，继续作为当前核心承接入口。

本轮仍不处理真实厂商背书、真实案例认证或最终资质信息。当前页面仍为预览版，真实资料、真实授权图片、最终二维码和联系方式需在正式上线前统一复核。

## V1.5 GitHub Pages 部署链路清理说明
当前版本已单独清理 GitHub Pages 部署链路。正式保留的部署文件为 `.github/workflows/pages.yml`，它会先生成 `_site`，再只把公网运行所需的首页静态文件发布到 GitHub Pages。

已删除重复的 `.github/workflows/static.yml`，避免把整个仓库、`custom/` 说明文档、截图、过程记录、甲方预览包和 GitHub workflow 配置一起发布到公网。此调整只影响部署链路，不改首页结构、视觉、文案或内容模块。

## V1.6 移动端导航修复说明
当前版本已针对手机端导航被遮盖、菜单按钮点不到、抽屉菜单层级不足和锚点滚动被 Header 遮挡的问题做了专项修复。

本次只修复移动端导航可用性：提高 Header、菜单按钮、遮罩和抽屉菜单层级；优化菜单打开后的滚动锁定；点击菜单项后先关闭菜单再滚动到对应栏目。首页视觉、Hero 结构、好物方案内容、推荐厂商内容和部署 workflow 均未调整。
