# 柚喜饰界官网上线前检查清单

文件说明：本文件用于“柚喜饰界｜柚木爱好者乐园”首页上线前逐项检查。
功能说明：覆盖内容、图片、二维码、SEO、移动端、CTA、授权和部署前确认。

结构概览：
  第一部分：内容与素材检查
  第二部分：页面体验检查
  第三部分：合规与部署检查

## 内容检查
- 确认首页没有把 demo 厂商当作真实厂商发布。
- 确认推荐厂商名称、城市、主营方向、案例和服务承诺均已由项目方确认。
- 确认好物方案内容为真实产品、真实推荐场景或真实空间方案，不展示价格，不出现“立即购买”。
- 确认柚木整装是否能对外承接，服务范围是否需要标注。
- 确认柚木地板是否能对外承接，产品参数、安装、交付和售后说明是否真实。
- 确认整装和地板相关文案没有“全国交付”“全案落地”“保证效果”“行业第一”等过度承诺。
- 确认柚木知识摘要与正式文章方向一致，不夸大、不写“行业第一”“全国领先”等表述。
- 确认 Footer 电话、邮箱、地址均为真实可公开信息。

## 图片检查
- 确认 Hero、知识、好物、推荐厂商图片均为真实授权素材。
- 确认图片无明显水印、无白底电商感、无过度滤镜。
- 确认图片在桌面端和手机端裁切后主体仍清晰。
- 确认所有 `<img>` 均有准确 `alt` 文案。
- 确认没有引用不可访问的远程图片。

## 二维码检查
- 扫码确认企业微信二维码可用。
- 扫码确认公众号二维码可用。
- 确认二维码对应账号、顾问、欢迎语和入群流程正确。
- 确认“备注：柚木社群／选购咨询／整装地板／厂商合作”与实际运营流程一致。

## 联系方式检查
- 拨打或核对官网电话。
- 发送测试邮件确认邮箱可用。
- 核对地址是否允许公开展示。
- 若地址只是产业园占位，必须上线前替换或删除。

## SEO 检查
- 确认 `title` 准确包含品牌、定位和核心栏目。
- 确认 `description` 不夸大，能说明知识平台、好物推荐、推荐厂商与社群入口。
- 确认 `keywords` 保留“推荐厂商”等核心词。
- 确认 Open Graph 标题、描述和图片存在。
- 确认页面只有一个核心 H1，H2/H3 层级合理。
- 确认导航名称全站统一为“认识柚喜｜柚木知识｜好物方案｜推荐厂商｜社群交流”。

## 移动端检查
- 手机端无横向溢出。
- 手机端“柚木知识”下拉分类可展开和收起，不遮挡整个页面。
- Hero 标题和“买柚木前，先来这里看一看。”清晰可读。
- 三个 Hero 按钮点击面积足够。
- 知识、好物、推荐厂商卡片单列展示时图片比例稳定。
- 企业微信二维码区域上下排列清楚，二维码不被压缩模糊。

## CTA 检查
- 首页所有导航、按钮、卡片、Footer 信息入口均可点击或明确不可点击，不进入不存在页面。
- 不存在 `href="#"`、空 `href` 或未处理的 `javascript:void(0)`。
- 下拉菜单里的所有分类均能滚动到对应内容位置。
- “添加柚喜顾问”能滚动到企业微信区。
- “加入柚木爱好者社群”能滚动到企业微信区。
- “了解柚木知识”能滚动到柚木知识区。
- “厂商合作咨询”能滚动到企业微信区。
- “领取《柚木选购避坑清单》”能滚动到企业微信区。
- “咨询柚木空间方案”“咨询柚木地板”“了解展示方向”“了解整装方向”“了解地板方向”“查看全部”不跳空、不报错。

## 法务与授权检查
- 图片、案例、厂商资料已取得展示授权。
- 不展示未经确认的真实客户名称、联系方式或隐私信息。
- 不写“权威认证”“官方认证”等可能引发责任的表述，除非已有正式证明。
- 如涉及用户反馈，需确认来源和公开授权。

## 备案与底部信息检查
- ICP 备案号已补充或确认暂不上线正式域名。
- 公安备案信息已确认是否需要展示。
- 版权年份、品牌名称、联系方式无误。
- 隐私政策、服务条款如需上线，需在正式发布前补齐。

## 部署前检查
- 本地预览可正常打开。
- 没有控制台明显报错。
- 没有不可访问远程资源。
- 压缩后的图片已替换到 `assets/`。
- 确认部署目标、域名、备案和缓存策略。
- 确认发布前已备份当前可运行版本。

## GitHub Pages 预览检查
- 确认 V1.3 GitHub Actions workflow `.github/workflows/pages.yml` 已存在。
- 确认 V1.5 后正式 Pages workflow 只保留 `.github/workflows/pages.yml`，不存在重复发布整个仓库的 `.github/workflows/static.yml`。
- 如果采用 Actions 发布，确认 `Settings -> Pages` 的 Source 为 `GitHub Actions`。
- 确认仓库 `Actions` 页面中的 `Deploy static site to GitHub Pages` 最近一次运行成功。
- 确认 workflow 发布目录为 `_site`。
- 确认 `_site` 只包含 `index.html`、`styles.css`、`script.js`、`assets/`、`data/`、`404.html`、`robots.txt`、`sitemap.xml`、`.nojekyll`。
- 确认 `custom/`、截图、项目记录、甲方预览包、`.github/` 和 `.git/` 不会发布到公网。
- 确认 GitHub Pages 是否已启用。

- 如果找不到 Pages，先确认当前账号是否有仓库 `Settings` 管理权限，再到 `Settings` 左侧 `Code and automation` 下查找 `Pages`。
- 确认预览地址 `https://yinuocheng123-cloud.github.io/youmu/` 可访问。
- 确认 `index.html` 位于仓库根目录。
- 确认 CSS、JS、图片、SVG 和 favicon 均使用相对路径。
- 确认不存在 `/assets/` 这类根路径资源引用。
- 确认 `.nojekyll` 已存在。
- 确认 `404.html` 可返回首页。
- 确认 `robots.txt` 当前是否仍为“不主动收录”策略。
- 确认 `sitemap.xml` 中的站点地址是否为最终预览或正式域名。
- 确认公网预览不包含购物车、价格、“立即购买”、登录、支付或数据库入口。
- 确认正式上线前是否需要删除或调整“预览版”提示。
